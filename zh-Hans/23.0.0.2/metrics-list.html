度量参考列表

度量参考列表包含可用于 Open Liberty的所有度量。 使用度量数据可有效监视微服务系统的状态。

度量从多个不同位置发出。 您可以从应用程序， Open Liberty 运行时和 Java 虚拟机 (JVM) 获取它们。 它们可以收集并存储在数据库工具 (例如 Prometheus) 中，并显示在仪表板 (例如 Grafana) 上。 有关将可观察性构建到应用程序中的更多信息，请参阅 具有度量的微服务可观察性。

启用 MicroProfile 指标 功能时，将始终报告一组基本指标。 您可以通过收集供应商度量 (可从 Open Liberty 服务器中的不同监视组件获取) 来扩充这组度量。 同时启用 性能监视 和 MicroProfile 度量功能时，将在 /metrics 端点上启用供应商度量报告。 "性能监视" 功能从所有可用的监视组件中检索统计数据。 MicroProfile Metrics 2.3 中引入的 REST 基本度量也依赖于性能监视功能。 在 MicroProfile Metrics 2.3之前，必须显式配置性能监视功能。 在 MicroProfile 度量 2.3和更高版本中， MicroProfile 度量在启动期间自动启用 "性能监视" 功能。

缺省情况下，将启用所有监视组件。 如果您的服务器收集的度量数据比您需要的更多，那么您可以通过仅收集您打算使用的那些供应商度量来提高服务器性能。 要仅配置要报告的供应商度量的子集，请在 server.xml 文件中的 monitor 配置元素的 filter 属性中指定要监视的组件。 您可以通过引用 可用于 Open Liberty 的度量 表的 监视组件 列来标识每个供应商度量的相关监视组件。

要仅启用 MicroProfile 度量值所使用的监视组件，请将以下代码添加到 server.xml 文件。

<monitor  filter="ConnectionPool,ThreadPool,RequestTiming,Session,WebContainer,REST,GrpcClient,GrpcServer"/>

要禁用所有供应商度量，但要保留 REST 基本度量，请配置 server.xml，如以下示例中所示:

<monitor  filter="REST"/>

要禁用所有监视组件，请将以下代码添加到 server.xml 文件:

<monitor  filter=" "/>

在下表中， Prometheus 度量名称列示在每个度量之后。 该表还列出了可用于 Open Liberty的所有度量的度量类型，度量单位和描述。 对于供应商度量值，还包含可用于过滤度量值的关联监视组件。 度量单位与度量名称和 Prometheus 名称一起包含，因为这些单位可能不同，因为 Prometheus 导出器会将度量缩放到基本单位。 例如，当 ft.<name>.bulkhead.executionDuration 度量在内部以纳秒为单位记录时， Prometheus 输出以秒为单位。 如果度量名称旁边没有列出任何单元，那么没有任何单元与该度量相关联。 表的 需要的功能部件 列包含一个或多个必须启用才能获取该度量数据的功能部件。 引入的版本 列指定必须启用才能收集度量值的功能部件的最低版本。
