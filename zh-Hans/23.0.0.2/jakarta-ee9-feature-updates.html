Jakarta EE 9.1 功能部件更新

针对 Jakarta EE 9.1 发行版更新了支持 Jakarta EE 平台的某些 Open Liberty 功能部件的名称和版本号。 在某些情况下，必须更改服务器配置中的功能部件版本和短名称，以更新 Jakarta EE 9.1的服务器。

在从 Java EE 到开放式源代码 Jakarta EE 平台的转换过程中，底层 API 包的名称空间从 javax 更改为 jakarta。 还会更新支持 Jakarta EE 平台的 Open Liberty 功能部件，以反映 Open Liberty 21.0.0.12发行版的此更改。 对于某些 Jakarta EE 功能部件，仅会更改版本号。 对于其他用户，将同时更改功能部件版本和短名称。 如果要将服务器更新到 Jakarta EE 9.1，那么必须在 server.xml 文件中指定更新的功能部件版本和短名称。

例如， "上下文和依赖关系注入" 功能将从 cdi-2.0 更新为 cdi-3.0。 要为 Jakarta EE 9.1更新此功能，仅需要更新版本号。 但是，在某些情况下，功能部件短名称也会更改，与 Enterprise JavaBeans 3.2 功能部件 (ejb-3.2) 一样，后者已更新为 Enterprise Beans 4.0 (enterpriseBeans-4.0)。 要更新此功能，必须同时指定新的版本号和新的短名称。

在以下 server.xml 文件示例中，指定了三个 Java EE 功能部件。

<server>
  <featureManager>
    <feature>servlet-4.0</feature>
    <feature>jpa-2.2</feature>
    <feature>jaxrs-2.1</feature>
  </featureManager>
</server>

要更新此配置以支持 Jakarta EE 9.1，必须指定更新的功能部件版本和短名称。 对于 servlet-4.0 功能部件，仅更改版本号。 但是，其他两个功能部件都更新了版本号和短名称。 jpa-2.2 功能部件变为 persistence-3.0 ， jaxrs-2.1 变为 restfulWS-3.0。 以下示例显示 Jakarta EE 9.1的已更新配置。

<server>
  <featureManager>
    <feature>servlet-5.0</feature>
    <feature>persistence-3.0</feature>
    <feature>restfulWS-3.0</feature>
  </featureManager>
</server>

对于具有已更新短名称的功能部件，如果使用 Jakarta EE 9.1 版本号指定旧短名称，那么服务器不会启动。 在这种情况下，服务器日志提供信息以指示必须指定哪些新功能部件短名称。 例如，如果 ejb-3.2 未正确更改为 ejb-4.0，那么服务器日志指示必须使用 enterpriseBeans-4.0 。

Open Liberty 提供了多个便利功能部件，这些功能部件可以同时启用一整套功能部件。 例如， Jakarta EE 平台 功能部件将启用支持 Jakarta EE 平台的所有功能部件。 如果将此功能部件从先前的短名称和版本号 javaee-8.0更新为 jakartaee-9.1，那么将自动启用所有 Jakarta EE 9.1 功能部件。 有关更多信息，请参阅 功能概述。
