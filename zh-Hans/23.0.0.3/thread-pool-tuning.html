线程池调整

Open Liberty 提供了自调整算法，用于控制其线程池的大小。 虽然您不需要为大多数应用程序手动调整线程池，但在某些情况下，您可能需要调整 coreThreads 和 maxThreads 属性。

Open Liberty 中的所有应用程序代码都在称为缺省执行程序的单个线程池中运行。 此池的大小由可管理各种工作负载的自调整控制器设置。 缺省执行程序池是运行应用程序代码的线程集。 Open Liberty 将其他线程用于诸如为 OSGi 框架提供服务，收集 JVM 垃圾以及提供 Java NIO 传输功能之类的任务。 但是，这些线程与应用程序性能不直接相关，并且大多数线程不可配置。

线程池调整行为

Open Liberty 线程池自调整进程每 1.5 秒运行一次。 线程池控制器从服务器启动时起维护一组关于线程池性能的数据。 吞吐量由控制器每个周期完成的任务数决定。 控制器记录先前尝试的各种池大小的吞吐量。 然后，将此历史吞吐量数据与当前周期的吞吐量进行比较，以确定最佳池大小。 在每个循环中，池大小可以递增或递减，也可以保持不变。

在某些情况下，没有可用于指导决策的历史数据。 例如，如果池在每个循环中都在增长，并且当前循环的大小是迄今为止最大的，那么对于较大的池大小，不存在有关吞吐量的数据。 在这种情况下，控制器会随机决定是否增加池的大小。 然后，根据该决定的结果对下一个周期进行重新调整。 此过程类似于人员线程池调谐器，该调谐器尝试各种线程池大小以查看它们的执行方式，并确定配置和工作负载的最佳值。

在每个 1.5-second 周期中，线程池控制器将通过以下自调整操作运行:

  1. 唤醒并检查池中的线程是否挂起。 如果任务在队列中，并且在上一个循环中未完成任何任务，那么控制器会将线程视为挂起。 在这种情况下，控制器会增加由设置指定的线程池大小，并跳过步骤 5。

  2. 使用在最近控制器周期中完成的任务数更新历史数据集。 性能记录为每个池大小的加权移动平均值。 此性能反映历史结果，但可根据不断变化的工作负载特征快速调整。

  3. 使用历史数据来预测在较小或较大的池大小时性能会更好。 如果不存在较小或较大的池大小的历史数据，那么线程池控制器将决定是增大还是缩小池大小。

  4. 在设置中指定的边界内增加或减少池大小，或者根据预测的性能使池大小保持不变。

  5. 回去睡觉

除线程池大小以外的各种因素会影响 Open Liberty 服务器中的吞吐量。 池大小与观察到的吞吐量之间的关系不是完全平滑或连续的。 因此，为了改进从历史吞吐量数据派生的预测，控制器不仅考虑最接近的更大和更小的池大小，还考虑每个方向的多个增量。

挂起解析

在某些应用程序场景中，池中的所有线程可能会被必须等待其他工作完成才能运行的任务阻塞。 在这些情况下，服务器可能会以特定池大小挂起。 要解决此情况，线程池控制器将进入挂起解决方式。

挂起解析会将线程添加到池中，以允许服务器恢复正常操作。 挂起解析还会缩短控制器循环持续时间，以快速打破死锁。

当控制器观察到正在再次完成任务时，将恢复正常操作。 控制器循环返回到其正常持续时间，并根据通常的吞吐量条件调整池大小。

手动线程池调整

在大多数环境，配置和工作负载中， Open Liberty 线程池不需要手动配置或调整。 线程池自调整以确定提供最佳服务器吞吐量所需的线程数。 线程池控制器在为 coreThreads 和 maxThreads 属性定义的边界内持续调整池中的线程数。 但是，在某些情况下，可能需要设置 coreThreads 或 maxThreads 属性。 以下部分描述了这些属性，并提供了可能需要手动调整这些属性的条件的示例。

  • coreThreads
    此属性指定池中的最小线程数。 此属性的最小值为 4。 Open Liberty 将为每个提供的工作创建一个新线程，直到线程数等于此属性的值为止。 如果未配置 coreThreads 属性，那么它缺省为可用于 Open Liberty 进程的硬件线程数的倍数。

    如果 Open Liberty 正在共享环境中运行，那么线程池控制器无法考虑与其共享可用 CPU 的其他进程。 在这些情况下，考虑到争用 CPU 资源的其他进程， coreThreads 属性的缺省值可能会导致 Open Liberty 创建的线程数超过最佳线程数。 在此情况下，可以将 coreThreads 属性限制为仅反映 Open Liberty 需要运行的 CPU 资源比例的值。

  • maxThreads
    此属性指定池中的最大线程数。 缺省值为 -1 ，等于 MAX_INT或实际上不受限制。

    某些环境会对进程可以创建的线程数设置硬限制。 目前， Open Liberty 无法知道这种上限是否适用，或者值是什么。 如果 Open Liberty 正在线程受限环境中运行，那么操作程序可以将 maxThreads 属性配置为可接受的值。

Open Liberty 线程池控制器旨在处理各种工作负载和配置。 在某些边缘情况下，您可能需要调整 coreThreads 和 maxThreads 属性。 但是，请先尝试缺省行为，以确保需要进行调整。
