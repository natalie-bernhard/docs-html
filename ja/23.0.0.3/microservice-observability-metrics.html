メトリックを使用したマイクロサービスの可観測性

Open Liberty は、 MicroProfile Metrics を使用して、多くの Open Liberty コンポーネントの内部状態を記述するメトリックを提供します。 MicroProfile Metrics は、 Open Liberty ランタイムおよびデプロイ済みアプリケーションによって発行されるすべてのメトリックにアクセスできる /metrics エンドポイントを提供します。

MicroProfile Metrics が有効になっていて、アプリケーションが実行されている場合、 /metrics エンドポイントにアクセスすることで、任意のブラウザーからメトリック・データを表示できます。 このエンドポイントの例は、 https://localhost:9443/metricsです。ここで、 9443 はアプリケーションのポート番号です。

/metrics エンドポイントで scope 照会パラメーターを使用してメトリック・レジストリーの有効範囲を指定することで、メトリックの有効範囲を絞り込むことができます。 例えば、 base メトリック・レジストリーのメトリック・データを表示するには、 /metrics?scope=base エンドポイントを使用します。 このパターンは、すべてのランタイム提供のスコープ (base、 application、および vendor) および任意の ユーザー定義メトリック・レジストリー・スコープにアクセスするために使用されます。

MicroProfile Metrics 4.0 以前では、ランタイム提供のスコープのみが使用可能であり、メトリック・データのスコープを絞り込むためにパス・パラメーターが使用されます。 エンドポイントは、 /metrics/base、 /metrics/application、および /metrics/vendorです。

メトリック・エンドポイントについて詳しくは、 MicroProfile メトリックおよびメトリック・エンドポイントを参照してください。

デフォルトでは、メトリック・データは Prometheus 形式で出力されます。 運用チームは、 Prometheusなどのツールを使用して、メトリックを収集し、データベースに保管することができます。 その後、 Grafanaなどのダッシュボードで分析のためにメトリック・データを視覚化できます。 バージョン 5.0以降、 MicroProfile Metrics は 組み込み Micrometer Metrics テクノロジー を使用するため、選択したサード・パーティーのモニタリング・ツールにメトリック・データを送信できます。

Open Libertyで使用可能なメトリックのリストについては、 メトリックのリファレンス・リストを参照してください。

アプリケーションへのメトリックの追加

MicroProfile Metrics with Open Liberty を使用するには、 server.xml ファイルで MicroProfile Metrics フィーチャーを使用可能にします。 アプリケーションにメトリックを追加するには、デフォルトのアプリケーション・レジストリーまたは独自のユーザー定義のメトリック・レジストリー・スコープを使用してメトリックを作成し、登録する必要があります。 メトリックは、登録されるとシステムに認識され、 /metrics エンドポイントから報告できます。 アプリケーションにメトリックを追加する最も簡単な方法は、アノテーションを使用することです。 MicroProfile Metrics は、コードにメトリックを素早くビルドするために使用できるアノテーションを定義します 。

ユーザー定義メトリック・レジストリー・スコープ

MicroProfile Metrics 5.0以降、デフォルトの application メトリック・レジストリー・スコープに加えて、メトリックを登録するための独自のメトリック・レジストリー・スコープを定義できます。 独自のスコープを定義するには、メトリック・アノテーションに scope パラメーターを指定するか、 RegistryScope アノテーションを使用して MetricRegistry クラスを注入し、 scope パラメーターを指定します。 メトリック・アノテーションまたは RegistryScope アノテーションで scope パラメーターの値を定義しない場合、デフォルトのスコープは applicationです。

以下の例では、 @Counted メトリック・アノテーションを使用して、 customScope ユーザー定義スコープを定義します。

@Counted(name="customCounter" description="This counter resides in a user-defined metric registry", scope="customScope")
public String countingThings() {
    return "1 2 3 4..!";
}

以下の例では、 customScope ユーザー定義スコープを使用して Java API でメトリックを計測する MetricRegistry クラスを注入します。

@Inject
@RegistryScope(scope="customScope")
MetricRegistry customMetricRegistry;

メトリック・タグ

MicroProfile メトリックでは、タグを使用してメトリックを計測することにより、類似した統計を追跡するメトリックを論理的にグループ化できます。 タグは、名前と値で構成されます。 同じタイプの複数のメトリックをグループ化するには、それらを同じ名前のタグで計測し、異なる値でグループ化します。

例えば、カウンターを使用する場合、タグによって識別されるメソッドのカウントを返す methodCounter という名前の複数のカウンターを作成できます。

@Counted(name="methodCounter", tags={"method=foo"})
public void foo() {
    //foo something
}

@Counted(name="methodCounter", tags={"method=bar"})
public void bar() {
    //bar something
}

ただし、 MicroProfile Metrics 5.0 以降では、使用されるタグ名のセットは、同じメトリック名を持つすべてのメトリック ID で同一でなければなりません。 そうでない場合は、 IllegalArgumentException がスローされます。 以下の例の最後の methodCounter インスタンスには、不整合な extraTag タグ名があり、 IllegalArgumentExceptionが生成されます。

@Counted(name="methodCounter", tags={"method=foo"})
public void foo() {
    //foo something
}

@Counted(name="methodCounter", tags={"method=bar"})
public void bar() {
    //bar something
}
@Counted(name="methodCounter", tags={"method=bad", "extraTag=bad"})
public void bad() {
    // bad!
}

以下のセクションでは、使用可能なメトリック・タイプと、それに対応するアノテーションの使用方法について説明します。 一部のメトリック・タイプは、 MicroProfile Metrics の特定のバージョンでのみ使用可能です。その他のメトリック・タイプは、 MicroProfile Metrics 5.0 の動作が以前のバージョンと若干異なる場合があります。 詳細は、各メトリック・タイプのセクションに記載されています。 詳しくは、 Differences between MicroProfile Metrics 5.0 and 4.0を参照してください。

  • タイマー

  • ヒストグラム

  • カウンター

  • ゲージ

  • メーター (MicroProfile Metrics 1.0-4.0 のみ)

  • 並行ゲージ (MicroProfile Metrics 2.0-4.0 のみ)

  • 単純タイマー (MicroProfile Metrics 2.3-4.0 のみ)

タイマー

タイマー・メトリックは、タイミング期間をナノ秒単位で集約し、期間統計を提供します。

@Timed アノテーションを使用して、コンストラクターまたはメソッドにタイマーのマークを付けます。 以下の例に示すように、タイマーは、アノテーション付きオブジェクトが開始される頻度と、呼び出しが完了するまでにかかる時間を追跡します。

@POST
@Path("/creditcard")
@Timed(
    name="donateAmountViaCreditCard.timer",
    description = "Donations that were made using a credit card")
public String donateAmountViaCreditCard(@FormParam("amount") Long amount, @FormParam("card") String card) {

    if (processCard(card, amount))
        return "Thanks for donating!";

    return "Sorry, please try again.";
}

MicroProfile Metrics 5.0以降、 mp.metrics.smallrye.timer.precision MicroProfile Config プロパティーを使用して、 Timer メトリックのパーセンタイル精度を調整できるようになりました。 このプロパティーは 1 から 5 までの値を受け入れ、値が指定されていない場合はデフォルトで 3 になります。 値を大きくすると、より正確なパーセンタイル計算が行われますが、メモリー・コストは大きくなります。 詳しくは、 MicroProfile Config プロパティー: MicroProfile Metricsを参照してください。

ヒストグラム

ヒストグラムは、値の分布を計算するメトリックです。 以下の情報が提供されます。

  • 最大値、中央値、および平均値

  • 50 番目、75 番目、95 番目、98 番目、99 番目、99.9 番目のパーセンタイルの値

  • 値の数

  • 値の標準偏差 (MicroProfile Metrics 1.0-4.0 のみ)

注: ヒストグラムの Prometheus形式のメトリック・データを表示しても、平均値は含まれません。

ヒストグラム・メトリックに注釈がありません。 ヒストグラムに値を記録するには、記録する値を指定して histogram.update(long value) メソッドを呼び出す必要があります。 ヒストグラムの現在の状態 (スナップショット) は、 getSnapshot() メソッドを使用して取得できます。 MicroProfile Metrics のヒストグラムでは、整数値または long 値のみがサポートされます。

以下の例は、寄付金の値を保管するために使用されるヒストグラムを示しています。 この例では、寄付金額の分布を管理者に示します。

Metadata donationDistributionMetadata = Metadata.builder()
              .withName("donationDistribution")                             // name
              .withDescription("The distribution of the donation amounts")  // description
              .withUnit("Dollars")                                          // units
              .build();
Histogram donationDistribution = registry.histogram(donationDistributionMetadata);
public void addDonation(Long amount) {
    totalDonations += amount;
    donations.add(amount);
    donationDistribution.update(amount);
}

カウンター

カウンター・メトリックは、増分カウントを保持します。 カウンターの初期値はゼロに設定され、アノテーションが付けられた要素が開始されるたびにメトリックが増分します。

@Counted アノテーションを使用して、メソッド、コンストラクター、またはタイプにカウンターのマークを付けます。 このカウンターは、注釈付きメソッドの合計呼び出し数をカウントして、単調増分します。

@GET
@Path("/no")
@Counted(name="no", description="Number of people that declined to donate.")
public String noDonation() {
    return "Maybe next time!";
}

ゲージ

ゲージ・メトリックを実装して、ゲージをサンプリングして特定の値を取得できるようにします。 例えば、ゲージを使用して CPU 温度やディスク使用量を測定することができます。

@Gauge アノテーションを使用して、メソッドをゲージとしてマーク付けします。

@Gauge(
    name="donations",
    description="Total amount of money raised for charity!",
    unit = "dollars",
    absolute=true)
public Long getTotalDonations(){
    return totalDonations;
}

メーター ( MicroProfile Metrics 1.0-4.0でのみ使用可能)

メーター・メトリックは、スループットを追跡します。 このメトリックは、以下の情報を提供します。

  • 平均スループット

  • 指数的に加重された移動平均スループット (1 分、5 分、および 15 分のマーク)

  • 測定値の数のカウント

@Metered アノテーションを使用して、コンストラクターまたはメソッドにメーターのマークを付けます。 メーターは、アノテーションが付けられたコンストラクターまたはメソッドの呼び出しをカウントし、それらが呼び出される頻度を追跡します。

@Metered(displayName="Rate of donations", description="Rate of incoming donations (the instances not the amount)")
public void addDonation(Long amount) {
    totalDonations += amount;
    donations.add(amount);
    donationDistribution.update(amount);
}

並行ゲージ ( MicroProfile Metrics 2.0-4.0でのみ使用可能)

並行ゲージ・メトリックは、アノテーションが付けられたエレメントの同時呼び出しをカウントします。 このメトリックは、各呼び出しの最高水準点と最低水準点も追跡します。 このメトリック・タイプは、 MicroProfile Metrics 5.0から削除されました。

@ConcurrentGauge アノテーションを使用して、メソッドに並行ゲージのマークを付けます。 並行ゲージは、アノテーションが付けられたメソッドが呼び出されると増分し、アノテーションが付けられたメソッドが戻ると減分し、アノテーションが付けられたメソッドの現在の呼び出しをカウントします。

@GET
@Path("/livestream");
@ConcurrentGauge(name = "liveStreamViewers", displayName="Donation live stream viewers", description="Number of active viewers for the donation live stream")
public void donationLiveStream() {
    launchLiveStreamConnection();
}

シンプル・タイマー ( MicroProfile Metrics 2.3-4.0でのみ使用可能)

単純なタイマー・メトリックは、経過したタイミング期間と呼び出しカウントを追跡します。 このタイプのメトリックは、 MicroProfile Metrics 2.3で使用可能です。 これは、 MicroProfile Metrics 5.0以降では削除されています。 単純タイマーは、パフォーマンスが高いタイマー・メトリックに代わる軽量のタイマーです。 MicroProfile Metrics 3.0以降、シンプル・タイマー・メトリックは、直前の完全な 1 分間に記録された最大および最小の期間も追跡します。 完全な分は、 00:00:00.000 秒から 00:00:59.999 秒として定義されます。

@SimplyTimed アノテーションを使用して、メソッド、コンストラクター、またはタイプに単純タイマーのマークを付けます。 シンプル・タイマーは、アノテーション付きオブジェクトが開始される頻度と、呼び出しが完了するまでにかかる時間を追跡します。

@GET
@Path("/weather");
@SimplyTimed(name = "weatherSimplyTimed", displayName="Weather data", description="Provides weather data in JSON")
public JSON getWeatherData() {
    retrieveWeatherData();
}
