Logstash 收集器事件参考列表

您可以使用 Logstash 收集器功能将日志事件发送到远程 Logstash 服务器，以便可以使用 Elasticsearch 和 Kibana 之类的产品对其进行管理和可视化。 每种类型的日志事件都有自己的一组字段，可用于定制 Kibana 仪表板。

Logstash 收集器 功能在运行时捕获日志事件，将这些事件分为多个字段，并将其安全地转发到配置的 Logstash 日志收集服务器。 有关更多信息，请参阅 使用 Logstash 收集器将日志和事件转发到 Logstash。

事件类型

Logstash 收集器功能部件生成以下事件类型:

  • 消息事件

  • 跟踪事件

  • HTTP 访问事件

  • 首次故障数据捕获 (FFDC) 事件

  • IBM JDK 的 垃圾回收事件

  • 审计事件

除了缺省日志和跟踪框架外， Logstash 收集器功能还会在启用二进制日志记录时转发消息事件和跟踪事件。 有关日志事件类型的更多信息，请参阅 日志管理。

  • EJB 委派 (SECURITY_AUTHN_委派)

  • 故障转移到基本认证 (SECURITY_AUTHN_FAILOVER)

  • 不受保护的 servlet 授权 (SECURITY_AUTHZ)

虚拟化环境中的 服务器和主机名

当 Open Liberty 服务器在容器或其他虚拟化环境中运行时，将根据配置中的特定变量自动设置 hostName 和 serverName 事件字段。

hostName 字段将自动设置为配置中可用的以下第一个值:

  • CONTAINER_HOST 环境变量的值

  • ${defaultHostName} Open Liberty 配置变量的值

  • JDK 报告的规范主机名

serverName 字段将自动设置为配置中可用的以下第一个值:

  • CONTAINER_NAME 环境变量的值

  • ${wlp.server.name} Open Liberty 配置变量的值

在容器中使用 Logstash 收集器功能时，可以在启动容器时设置 CONTAINER_HOST 和 CONTAINER_NAME 环境变量。 设置这些环境变量可确保 Logstash 收集器功能部件使用相应的主机和容器名称对其发送的记录进行标记，这有助于确定问题。 如果未设置这些环境变量，那么在使用显示来自多个容器的事件的仪表板时，您可能无法确定事件源自哪个容器。

启动容器时，可以使用类似于以下示例的命令来设置这些环境变量:

docker run -d -e LICENSE=accept -e CONTAINER_NAME=yourContainerName -e CONTAINER_HOST=yourContainerHost --name=yourContainerName yourImageName

如果使用 Podman 来管理容器，请运行以下命令:

podman run -d -e LICENSE=accept -e CONTAINER_NAME=yourContainerName -e CONTAINER_HOST=yourContainerHost --name=yourContainerName yourImageName

当 Open Liberty 服务器在 IBM Cloud® Kubernetes Service中运行时，已为您设置 CONTAINER_HOST 和 CONTAINER_NAME 环境变量。
