JDBC データ・ソースの Kerberos 認証

Kerberos は、クライアントとサーバーが、信頼できる第三者の鍵配布センター (KDC) と通信することによって相互に認証できるネットワーク認証プロトコルです。 Kerberos 資格情報を使用して、Java Database Connectivity (JDBC) データ・ソースによってバッキングされるデータベースを認証するように、 Open Liberty サーバーを構成することができます。

Open Liberty サーバーで実行されるアプリケーションは、ユーザー・アカウント情報や販売トランザクション・レコードなど、データベース内の保護リソースにアクセスする必要がある場合があります。 そのようなデータベースが Kerberos 認証によって保護されている場合、 Open Liberty サーバーは、保護リソースにアクセスし、データベースの認証性を確認するために必要な資格情報を提供することができます。

JDBC データ・ソース用の Kerberos 構成

server.xml ファイルで、 Kerberos 資格情報を使用する任意の Open Liberty フィーチャーに値を提供する Kerberos 構成を指定できます。 この構成はオプションです。 詳しくは、 Open Liberty の Kerberos 認証を参照してください。

JDBC データ・ソース用に Kerberos 認証を構成するには、 Java Database Connectivity フィーチャーを使用可能にする必要があります。 JDBC データ・ソース接続について詳しくは、 データ・ソース構成を参照してください。 Kerberos 構成を定義した後、以下の例に示すように、 server.xml ファイル内の authData エレメントで Kerberos プリンシパルを構成できます。

<authData id="myKerberosAuth" krb5Principal="krbUser" krb5TicketCache="${server.config.dir}/security/krb5cc_krbUser"/>

<library id="db2DriverLib">
  <fileset dir="${server.config.dir}/db2"/>
</library>

<dataSource jndiName="jdbc/krb/basic" containerAuthDataRef="myKerberosAuth">
  <jdbcDriver libraryRef="${server.config.dir}/db2DriverLib"/>
  <properties.db2.jcc databaseName="${DB2_DBNAME}" serverName="${DB2_HOSTNAME}" portNumber="${DB2_PORT}"/>
</dataSource>

krb5TicketCache 属性はオプションです。 この属性が指定されていない場合、 Open Liberty は、 kerberos エレメントで構成されている keytab ファイルから資格情報値を解決します。 keytab ファイルが構成されていない場合、 Open Liberty は、Java SDK のデフォルトの Kerberos 資格情報キャッシュ・ロケーションから資格情報値を解決します。 詳しくは、 Open Liberty の Kerberos 認証を参照してください。

containerAuthDataRef 属性は、データベース接続を認証するために authData エレメントを参照します。

構成の移植性を有効にするために、 databaseName、 serverName、および portNumber プロパティーの値は変数によって提供されます。 詳しくは、 変数置換の優先順位を参照してください。

オプションで、 dataSource エレメントに recoveryAuthDataRef 属性を指定して、XA リカバリー時の認証に krb5Principal 属性および krb5TicketCache 属性の代替値を使用することができます。 recoveryAuthDataRef 属性が指定されていない場合、 containerAuthDataRef 属性によって参照される authData エレメントが、XA リカバリー時の認証に使用されます。

Oracle JDBC シン・ドライバー ・バージョン 21c 以前は、 Oracle JDK または OpenJDK Kerberos 実装のみをサポートしています。 したがって、これらのバージョンを IBM SDK Java Technology Edition バージョン 8で使用することはできません。 IBM Java 8 でのデータ・ソースの Kerberos 認証をサポートするために、バージョン 21c 以降の Oracle JDBC シン・ドライバーに更新します。

SPNEGO 認証との互換性

Open Liberty は、Web 認証および JDBC データ・ソース認証のための SPNEGO 認証 もサポートします。 kerberos エレメントはオプションですが、SPNEGO 認証に使用される keytab ファイルおよび configFile ファイルの場所をカスタマイズするために使用できます。

JDBC データ・ソースで SPNEGO 認証を使用するために、 datasource エレメントで containerAuthDataRef 属性または recoveryAuthDataRef 属性を構成する必要はありません。 ただし、SPNEGO 認証で XA 対応データ・ソースを使用する場合は、自動 XA リカバリーに SPNEGO 認証を使用できないため、データ・ソースに SPNEGO 以外の recoveryAuthDataRef 値を構成する必要があります。

例えば、以下の構成では、 JDBC データ・ソースで IBM Db2 データ・ソースに対する SPNEGO 認証を使用できます。

<featureManager>
  <feature>jdbc-4.2</feature>
  <feature>spnego-1.0</feature>
  <feature>appSecurity-2.0</feature>
</featureManager>

<spnego servicePrincipalNames="HTTP/my.spnego.server.com" authFilterRef="myAuthFilter"/>
<authFilter id="myAuthFilter">
  <requestUrl urlPattern="/MySpnegoServlet" matchType="contains"/>
</authFilter>

<kerberos keytab="${server.config.dir}/security/krb5.keytab" configFile="${server.config.dir}/security/krb5.conf"/>

<library id="db2DriverLib">
  <fileset dir="${server.config.dir}/db2"/>
</library>

<dataSource jndiName="jdbc/spnegoAuth">
  <jdbcDriver libraryRef="db2DriverLib"/>
  <properties.db2.jcc databaseName="${DB2_DBNAME}" serverName="${DB2_HOSTNAME}" portNumber="${DB2_PORT}"/>
</dataSource>

も参照してください。
