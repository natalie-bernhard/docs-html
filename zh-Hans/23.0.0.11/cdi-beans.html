上下文和依赖关系注入 (CDI)

上下文和依赖关系注入 (CDI) 是一个使集成不同类型的 Jakarta EE 组件更容易的框架。 CDI 支持自动注入 Java 对象依赖关系，并帮助管理这些依赖关系的生命周期。

CDI 提供以下基本服务:

  • 上下文，可帮助您将有状态组件的生命周期和交互绑定到定义良好的可扩展上下文。

  • 依赖关系注入，可帮助您以类型安全的方式将组件注入到应用程序中。

CDI 是 MicroProfile， Java EE和 Jakarta EE中的基本元素。 MicroProfile 规范采用 CDI 优先方法。 它们依赖于 CDI 扩展机制，该机制实现了更多功能，并以新的能力增强了现有标准。

您可以通过启用 Jakarta Contexts and Dependency Injection 功能部件将 CDI 与 Open Liberty 应用程序配合使用。

CDI Bean

Bean 是可以通过依赖关系注入相互注入的组件。 几乎每个具有构造函数的纯旧 Java 对象 (POJO) 都可以是 Bean。 在 CDI 中， bean 是定义应用程序状态和逻辑的上下文对象的源。 这些 Bean 由 CDI 容器通过使用 Bean 定义注释进行管理。 容器会实例化依赖关系，并控制何时以及如何实例化和销毁组件。

注释是有关代码的元数据。 您可以使用它们来装饰类，方法，字段，参数，变量，构造函数或包。 您还可以使用注释来定义存储 Bean 的作用域。

CDI 作用域

作用域定义 Bean 的状态。 以下 CDI 作用域注释控制 Bean 的生命周期:

  • @ApplicationScoped 类级别注释指定在应用程序处于活动状态时实例处于活动状态。

  • @RequestScoped 类级别注释指定为每个请求 (例如 servlet 请求) 创建 Bean 的新实例。

  • @Dependent 类级别注释指定新实例属于另一个对象。 从属 Bean 实例从不在不同客户机或不同注入点之间共享。 创建它所属的对象时将其实例化，然后销毁它所属的对象时将其销毁。

如果未定义任何类级别注释，那么缺省值为 @Dependent 注释。

必须发现定义 CDI Bean 的注释。 您可以使用 .jar 文件的 META-INF 文件夹或 .war 文件的 WEB-INF 文件夹中的 beans.xml 文件来启用 CDI 注释扫描。 此文件可以为空，也可以包含类似于以下示例的代码:

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns="https://jakarta.ee/xml/ns/jakartaee"
    xsi:schemaLocation="https://jakarta.ee/xml/ns/jakartaee https://jakarta.ee/xml/ns/jakartaee/beans_4_0.xsd"
    version="4.0"
    bean-discovery-mode="annotated">
</beans>

在 CDI 3.0 和更低版本中，存在空的 beans.xml 文件或具有 bean-discovery-mode="all" 属性的 beans.xml 文件会生成所有潜在对象 CDI Bean。 在其他情况下，只有具有 CDI Bean 定义注释的对象才是 CDI Bean。

在 CDI 4.0 和更高版本中，存在空的 beans.xml 文件会导致将归档视为隐式 Bean 归档，相当于设置 bean-discovery-mode="annotated" 属性。 只有具有 CDI bean 定义注释的对象才是 CDI bean。 如果需要，可以设置 cdiemptyBeansXmlCDI3Compatibility 属性以启用与先前版本的兼容性。 在 CDI 4.0 中将此属性设置为 true 会导致将空 beans.xml 文件视为显式 bean 归档。

同样在 CDI 4.0 和更高版本中，未指定 bean-discovery-mode 属性的非空 beans.xml 文件会将此值缺省为 annotated。 因此，缺省情况下，只有具有 CDI bean 定义注释的对象才会发现为 CDI bean。 但是，请始终正确指定所有 beans.xml 文件的版本，如上例中所示。

CDI bean 注入

CDI 通过 @Inject 注释将定义的 Bean 注入到其他组件中。 例如，以下示例中的 POJO 是 CDI bean。 由于 MyBean 类具有 @ApplicationScoped 注释，因此仅创建一个实例。 您可以在应用程序的整个生命周期内使用同一实例。

@ApplicationScoped
public class MyBean {
    int i=0;
    public String sayHello() {
        return "MyBean hello " + i++;
    }
}

在以下示例中， MyRestEndPoint 类指定 @RequestScoped 注释，这表示为每个请求创建一个实例。 CDI 将相同的 MyBean 实例插入到每个请求中:

@RequestScoped
@Path("/hello")
public class MyRestEndPoint {
    @Inject MyBean bean;
    @GET
    @Produces (MediaType.TEXT_PLAIN)
    public String sayHello() {
        return bean.sayHello();
    }
}
