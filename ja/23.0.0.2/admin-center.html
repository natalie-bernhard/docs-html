Admin Center を使用した Open Liberty の管理

Admin Center は、 Open Liberty サーバー用の Web ベースのグラフィカル・ユーザー・インターフェース (GUI) です。 Admin Center を使用して、サーバー構成、 OpenID Connect 許可、および Java バッチ・ジョブを管理できます。

Admin Center のセットアップ

  1. ランタイムに Admin Center フィーチャーがまだインストールされていない場合は、以下の例に示すように、 path_to_liberty/wlp/bin ディレクトリーから featureUtility installFeature コマンドを実行してインストールします。

    featureUtility installFeature adminCenter-1.0

  2. server.xml ファイルで Admin Center フィーチャー を有効にし、リーダーまたは管理者のいずれかの役割に対して 管理役割を構成 します。

  3. Admin Center は HTTPS 経由でのみアクセス可能です。HTTPS ポートを指定し、サーバー認証資格情報を含む keystore ファイルを保護するように鍵ストアを構成する必要があります。 以下のサンプル server.xml ファイルを使用して、Admin Center のテスト・インスタンスをセットアップできます。

この例では、 adminCenter-1.0 フィーチャーを有効にし、 reader-role および administrator-role の管理役割でユーザーを定義する 基本ユーザー・レジストリー を構成します。 また、 httpEndpoint エレメントに HTTPS ポートを指定し、 keyStore エレメントにデフォルトの鍵ストアとパスワードの参照を指定します。

<server description="Admin Center example server">
   <featureManager>
      <feature>adminCenter-1.0</feature>
   </featureManager>
   <basicRegistry id="basic">
      <user name="admin" password="adminpwd" />
      <user name="reader" password="readerpwd" />
   </basicRegistry>
   <administrator-role>
      <user>admin</user>
   </administrator-role>
   <reader-role>
      <user>reader</user>
   </reader-role>
   <keyStore id="defaultKeyStore" password="Liberty" />
   <httpEndpoint id="defaultHttpEndpoint"
                 host="*"
                 httpPort="9080"
                 httpsPort="9443" />
</server>

Admin Center にログイン ( L)

Admin Center GUI にアクセスする前に、サーバーが稼働していることを確認してください。 Web ブラウザーで、以下の URL の Admin Center ログイン・ページに移動します。

https://host_name:port_number/adminCenter

host_name 値には、 Open Liberty サーバーが実行されているコンピューターの IP アドレスまたはドメイン・ネーム・サーバー (DNS) ホスト名を指定します。 ブラウザーがサーバーと同じコンピューター上で実行されている場合は、ホスト名として localhost を指定できます。 port_number 値には、 server.xml ファイル内の httpEndpoint エレメントから httpsPort 値を指定します。

例えば、ご使用のブラウザーがサーバーと同じコンピューター上で実行されていて、前の例の server.xml 構成を使用している場合、Admin Center には以下の URL からアクセスできます。

https://localhost:9443/adminCenter

接続が信頼できることを確認するプロンプトがブラウザーに表示された場合は、例外を指定するか、Admin Center への接続を続行できるようにします。 Admin Center にログインするには、管理ロール構成からユーザー名とパスワードを指定します。 例えば、以前に提供されたサンプルの server.xml 構成を使用する場合は、ユーザー名として admin を指定し、パスワードとして adminpwd を指定します。

以下の画面キャプチャーは、 「Admin Center ログイン (Admin Center login)」 画面を示しています。

ツールボックスからツールを選択します

Admin Center にログインすると、ブラウザーに 「ツールボックス」が表示されます。このツールボックスには、 Server Config および Explore ツールなどのツールと、 Open Liberty Web サイトへのブックマークが含まれています。

以下の画面キャプチャーは、Admin Center の 「ツールボックス」を示しています。

「サーバー構成」 および 「探索」 ツールは、デフォルトで使用可能です。 Java バッチ ・ツールおよび OpenID Connect ツールは、これらの機能に対してそれぞれの Open Liberty フィーチャーを使用可能にして構成した場合にのみ使用可能です。 「ツールボックス」 からツールを追加または削除するには、 「編集」 アイコンを選択します。 ツールを使用するには、その 「ツールボックス」 アイコンを選択します。

以下のツールが Admin Center から使用可能です。

  • サーバー構成

  • 体験

  • Java バッチ

  • OpenID Connect ツール

サーバー構成ツールを使用したサーバー構成の管理およびテスト

「サーバー構成」 ツールを使用して、 Open Liberty サーバー構成ファイルを表示および編集したり、さまざまなリソースへのサーバー接続をテストしたりすることができます。 構成ファイルまたはテスト接続を編集するには、 administrator-role 管理役割を持っている必要があります。

構成ファイルを編集する前に、ツールにより、 remoteFileAccess エレメントを server.xml ファイルに追加するよう求めるプロンプトが出されます。 サンプル要素は、ブラウザー・ウィンドウからコピーすることも、手動で構成することもできます。 このエレメントを追加しない場合、構成ファイルは読み取り専用モードで表示されます。

構成ファイルへの書き込みアクセスを有効にするには、以下のコードを server.xml ファイルに追加します。

<remoteFileAccess>
   <writeDir>${server.config.dir}</writeDir>
</remoteFileAccess>

「サーバー構成」 ツールは、 server.xml ファイルなどの構成ファイルを、 「設計」 と 「ソース」の 2 つのモードで表示します。 これらの各モードには、ツール内のブラウザー・タブからアクセスできます。 これらのモードをカスタマイズするには、 「設定」 アイコンをクリックします。 例えば、 デザイン ・モードでパラメーターの説明を追加または削除したり、 ソース ・モードで行番号を追加または削除したりすることができます。

「設計」 モードでは、インライン文書でグラフィカル・コントロールを使用して構成ファイルの内容が表示されます。 このモードでは、構成内のエレメントを選択して、使用可能なフィーチャーおよびパラメーター設定を表示します。

「ソース」 モードでは、ファイル・テキストを編集するための直接アクセスが可能になり、ファイル・エレメントのツールチップの説明が提供されます。 要素とパラメーターの上にカーソルを移動すると、その文書が表示されます。 Ctrl+Space を押すとコンテンツ・アシストを使用できます。コンテンツ・アシストは、新しいエレメント、パラメーター、および値の追加を支援します。 ただし、 「ソース」 モード・エディターでは、指定した構成値は検証されません。 「ソース」 モードで行った変更は、実行中のサーバーに適用されます。 無効な構成値を指定すると、エラーが発生したり、サーバーに障害が発生したりする可能性があります。

以下の画面キャプチャーは、 「サーバー構成」 ツールの 「設計」 タブを示しています。

サーバー構成ツールを使用したリソースへのサーバー接続のテスト

サーバー構成ツールを使用して、Java Database Connectivity (JDBC)、 Java EE Connector Architecture (JCA)、Java Message Service (JMS)、および Cloudant リソースへの接続をテストすることができます。

  1. 「サーバー構成」 ツールを使用して接続をテストするには、 Admin Center フィーチャーに加えて、 server.xml ファイルで Admin REST Connector および MicroProfile OpenAPI フィーチャーを有効にする必要があります。 また、接続を確認するサーバー・リソースをサポートするためのフィーチャーも有効にする必要があります。 以下の例では、データベース接続をサポートするために JDBC フィーチャーも使用可能にします。

    <featureManager>
   <feature>adminCenter-1.0</feature>
   <feature>restConnector-2.0</feature>
   <feature>mpOpenApi-3.0</feature>
   <feature>jdbc-4.3</feature>
   ...
</featureManager>

  2. 「サーバー構成」 ツールで、 「設計」 タブの「サーバー」メニューから、接続をテストするリソースを選択します。

  3. 選択したリソースの表示ウィンドウで、 「テスト」 ボタンをクリックします。

    以下の画面キャプチャーは、 「設計」 タブと、 「サーバー構成」 ツールで選択したリソースの表示ウィンドウを示しています。

  4. 接続テスト用に使用する認証方式を選択します。 3 つの認証方式は、 「コンテナー認証」、 「アプリケーション認証」、または 「リソース参照なし」です。

    以下の画面キャプチャーは、 「接続テスト」 ページと認証方式を示しています。

  5. 選択した認証方式で、必須フィールドを指定し、 「接続テスト」 ボタンをクリックします。 テストの結果が表示されます。

Admin Center は、REST API を使用して接続を検証します。 詳しくは、 サーバー接続の検証を参照してください。

探索ツールを使用したリソースのモニターおよび管理

「探索」 ツールを使用して、 Open Liberty リソースをモニターおよび管理できます。 このツールは、サーバーとそのアプリケーションに関する情報を表示するためのオプションを提供します。 また、これを使用して、リソースを停止、開始、または再始動することもできます。

「探索」 ツールでサーバーおよびアプリケーションの状況をよりタイムリーに更新するには、 Web ソケット 機能をサーバー構成に追加します。 この機能は、サイズに関係なく、トポロジーのライブ・ビューを提供します。 Web ソケット機能を使用しないと、Admin Center は定期的かつ頻繁に変更をポーリングします。

ナビゲーション・バーから 「アプリケーション」 ビューを選択して、サーバー上で実行されているアプリケーションをモニター、停止、開始、または再始動することができます。

ナビゲーション・バーの 「モニター」 ビューには、サーバーとシステムのメトリックがグラフでグラフィカルに表示されます。 「モニター」ビューを使用して、ヒープ・メモリー、ロードされたクラス、アクティブな Java 仮想マシン (JVM) スレッド、および中央演算処理装置 (CPU) 使用量を追跡できます。

以下の画面キャプチャーは、 「探索」 ツールの 「モニター」 ビューを示しています。

「モニター」 ビューにさらにリソース・メトリックを追加するには、 「編集」 アイコンを選択し、メニューからオプションを選択します。 「パフォーマンス・モニター」 機能が有効になっている場合、このメニューからさらに多くのリソース・メトリックを使用できます。 サーバーに構成されているリソースによっては、以下のリソース・メトリックが使用可能な場合があります。

  • アクティブ・セッション

  • アクティブ Liberty スレッド

  • 平均応答時間

  • 平均待ち時間

  • 要求カウント

  • 使用接続数

Java バッチ・ツールを使用した Java バッチ・ジョブの管理

バッチ管理 フィーチャーを構成すると、 Java バッチ ・ツールにアクセスできます。 このツールを使用して、Java バッチ・ジョブの進行状況と状況の表示、そのインスタンスの管理、およびログ・ファイルの表示を行うことができます。

以下の画面キャプチャーは、 Java バッチ ・ツールを示しています。

各バッチ・ジョブには、ジョブを停止、再始動、またはパージするために選択できる 「アクション」 アイコンと、ジョブ・ログを表示するために選択できる 「ログの表示」 アイコンがあります。 バッチ・ジョブの詳細およびメトリックを表示するには、 「バッチ・ジョブ名」 の上にカーソルを移動し、ツールチップ・メニューからオプションを選択します。

バッチ・ジョブまたはジョブ・ログがリモート・サーバー上にある場合は、各リモート・サーバーで クロス・オリジン領域共有 (CORS) を構成します 。 CORS により、Admin Center はリモート・サーバーからジョブ情報を要求できます。

バッチ管理機能には、バッチ・ジョブを表示および管理するためのカスタム許可が必要です。 Java バッチ・ツールを使用するには、Admin Center へのアクセスに必要なリーダーまたは管理者の管理ロールに加えて、 com.ibm.ws.batch カスタム許可ロールを構成する必要があります。 以下の server.xml ファイルの例は、管理者管理ロールおよび batchAdmin カスタム許可ロールを付与された wanda ユーザーの構成を示しています。

  <administrator-role>
      <user>wanda</user>
  </administrator-role>

  <authorization-roles id="com.ibm.ws.batch">
      <security-role name="batchAdmin">
          <user name="wanda"/>
      </security-role>
  </authorization-roles>

この構成では、構成された Java バッチ・ジョブを表示および管理する権限がユーザーに付与されます。

com.ibm.ws.batch カスタム許可ロールは、リーダー管理ロールと組み合わせることもできます。 この組み合わせにより、Java バッチ・ツールへのフルアクセスは引き続き許可されますが、他の Admin Center リソースへの読み取り専用アクセスは提供されます。

ただし、カスタム・バッチ許可ロールが構成されていない場合、管理者管理ロールのユーザーであっても、Java バッチ・ジョブを表示したり管理したりすることはできません。

OpenID Connect (OIDC) ツールを使用した OpenID Connect プロバイダー・タスクの管理

OpenID Connect Provider フィーチャー を有効にし、 OpenID Connectを構成すると、Admin Center から以下の OpenID Connect ツールにアクセスできます。

OpenID Connect クライアント管理

このツールを使用して、管理者が OIDC プロバイダー上のクライアントを管理できるようにすることができます。

OpenID Connect パーソナル・トークン管理

このツールを使用して、ユーザーが OIDC プロバイダーでアプリケーション・パスワードおよびアプリケーション・トークンを管理できるようにすることができます。

OpenID Connect ユーザー・トークン管理

このツールを使用して、管理者が OIDC プロバイダー上の他のユーザーのアプリケーション・パスワードおよびアプリケーション・トークンを取り消すことができます。

詳しくは、 Admin Center OpenID Connect ツールを参照してください。
