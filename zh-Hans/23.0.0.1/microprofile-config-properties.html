MicroProfile 配置属性

MicroProfile Config 是一个 API ，它使用上下文和依赖关系注入 (CDI) 将配置属性值直接注入到应用程序中，而无需应用程序代码来检索这些值。 您可以使用 MicroProfile 配置属性为其他 MicroProfile API 提供配置值。

有关 MicroProfile Config 的更多信息，请参阅 Microservices 的外部配置 和 Open Liberty MicroProfile Config 指南。

以下部分提供了列出和描述可用于不同 MicroProfile API 的 MicroProfile 配置属性的表。

  • MicroProfile JSON Web 令牌

MicroProfile 上下文传播

启用 MicroProfile 上下文传播时，可以指定以下 MicroProfile 配置属性。 有关更多信息，请参阅 MicroProfile 上下文传播规范。

MicroProfile 上下文传播的 MicroProfile 配置属性
名称 描述 当运行时读取属性时 示例

mp.context.ManagedExecutor.cleared

指定 ManagedExecutor 类的 cleared 参数的缺省值。

当 ManagerExecutor.builder() 类用于构建 ManagedExecutor 实例并且构建器未指定已清除的上下文类型集时。

mp.context.ManagedExecutor.cleared=Remaining

mp.context.ManagedExecutor.maxAsync

指定 ManagedExecutor 类的 maxAsync 参数的缺省值。

当 ManagerExecutor.builder() 类用于构建 ManagedExecutor 实例并且构建器未指定 maxAsync 属性时。

mp.context.ManagedExecutor.maxAsync=10

mp.context.ManagedExecutor.maxQueued

指定 ManagedExecutor 类的 maxQueued 参数的缺省值。

当 ManagerExecutor.builder() 类用于构建 ManagedExecutor 实例并且构建器未指定 maxQueued 属性时。

mp.context.ManagedExecutor.maxQueued=1

mp.context.ManagedExecutor.propagated

指定 ManagedExecutor 类的 propagated 参数的缺省值。

当 ManagerExecutor.builder() 类用于构建 ManagedExecutor 实例并且构建器未指定传播的上下文类型集时。

mp.context.ManagedExecutor.propagated=Security,Application

mp.context.ThreadContext.cleared

指定 ThreadContext 类的 cleared 参数的缺省值。

当 ThreadContext.builder() 类用于构建 ThreadContext 实例并且构建器未指定已清除的上下文类型集时。

mp.context.ManagedExecutor.cleared=Security,Transaction

mp.context.ThreadContext.propagated

指定 ThreadContext 类的 propagated 参数的缺省值。

当 ThreadContext.builder() 类用于构建 ThreadContext 实例并且构建器未指定传播的上下文类型集时。

mp.context.ThreadContext.propagated=None

mp.context.ThreadContext.unchanged

指定 ThreadContext 类的 unchanged 参数的缺省值。

当 ThreadContext.unchanged() 类用于构建 ThreadContext 实例并且构建器未指定已清除的未更改类型集时。

mp.context.ManagedExecutor.unchanged=Remaining

MicroProfile 容错

启用 MicroProfile 容错时，可以指定以下 MicroProfile 配置属性。 有关更多信息，请参阅 MicroProfile 容错规范。

MicroProfile 配置 MicroProfile 容错属性
名称 描述 当运行时读取属性时 示例

classname/methodname/annotationname/parametername=newvalue

覆盖在方法级别的容错注释中设置的值。

在 MicroProfile Fault Tolerance V 3.0 及更高版本中，在第一次执行此方法时，将对每个 Bean 类的每个方法读取此属性一次。 在 MicroProfile Fault Tolerance V 2.1 和更低版本中，每个 Bean 实例的每个方法在第一次执行该方法时读取一次。

com.acme.ClassA/methodB/Retry/maxRetries=5

classname/annotationname/parametername=newvalue

覆盖在注释级别的容错注释中设置的值。

在 MicroProfile Fault Tolerance V 3.0 及更高版本中，在第一次执行此方法时，将对每个 Bean 类的每个方法读取此属性一次。 在 MicroProfile Fault Tolerance V 2.1 和更低版本中，每个 Bean 实例的每个方法在第一次执行该方法时读取一次。

com.acme.ClassA/Retry/maxRetries=5

mp.fault.tolerance.interceptor.priority

配置容错拦截器基本优先级。 此属性在 MicroProfile Fault Tolerance V 2.0 和更高版本中可用。

应用程序启动

mp.fault.tolerance.interceptor.priority=1000

MP_Fault_Tolerance_Metrics_Enabled

禁用或启用由容错生成的所有度量。 缺省值为 true。 此属性在 MicroProfile Fault Tolerance V 1.1 和更高版本中可用。

应用程序启动

MP_Fault_Tolerance_Metrics_Enabled=false

MP_Fault_Tolerance_NonFallback_Enabled

禁用或启用除 @Fallback以外的所有容错注释。 缺省值为 true。

在 MicroProfile Fault Tolerance V 3.0 及更高版本中，在第一次执行此方法时，将对每个 Bean 类的每个方法读取此属性一次。 在 MicroProfile Fault Tolerance V 2.1 和更低版本中，每个 Bean 实例的每个方法在第一次执行该方法时读取一次。

MP_Fault_Tolerance_NonFallback_Enabled=false

MicroProfile GraphQL

启用 MicroProfile GraphQL 时，可以指定以下 MicroProfile 配置属性。 有关更多信息，请参阅 MicroProfile GraphQL 规范。

MicroProfile MicroProfile GraphQL
名称 描述 当运行时读取属性时 示例

mp.graphql.defaultErrorMessage

设置缺省错误消息。

当应用程序抛出未检查的异常时。

mp.graphql.defaultErrorMessage=Unexpected failure in the system. Jarvis is working to fix it.

指定要包含的未检查异常消息的允许列表 (以逗号分隔)。 缺省情况下，所有未检查的异常都在阻止列表中。

当应用程序抛出未检查的异常时。

mp.graphql.exceptionsWhiteList=org.eclipse.microprofile.graphql.tck.apps.superhero.api.WeaknessNotFoundException

mp.graphql.exceptionsBlackList

隐藏已检查异常的消息。 缺省情况下，所有已检查的异常都在允许列表中。

当应用程序抛出已校验的异常时。

mp.graphql.exceptionsBlackList=java.io.IOException,java.util.concurrent.TimeoutException

MicroProfile JSON Web 令牌

启用 MicroProfile JSON Web 令牌 (JWT) 时，可以指定以下 MicroProfile 配置属性。 有关更多信息，请参阅 MicroProfile JSON Web 令牌规范。

MicroProfile JSON Web 令牌的 MicroProfile 配置属性
名称 描述 当运行时读取属性时 示例

mp.jwt.decrypt.key.location

指定密钥管理密钥的位置，这是用于解密内容加密密钥的专用密钥。 然后，内容加密密钥用于解密 JSON Web 加密 (JWE) 密文。 这个专用密钥必须对应于用来将内容加密密钥解密的公用密钥。 此属性在 MicroProfile JSON Web Token 1.2 和更高版本中可用。

mp.jwt.token.cookie

指定期望包含 JWT 令牌的 cookie 的名称。 缺省值为 Bearer。 此属性在 MicroProfile JSON Web Token 1.2 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.token.cookie=myToken

mp.jwt.token.header

控制期望包含 JWT 令牌的 HTTP 请求头。 有效值为 Authorization(缺省值) 或 Cookie。 此属性在 MicroProfile JSON Web Token 1.2 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.token.header=Cookie

mp.jwt.verify.audiences

指定允许的 aud 值的列表。 必须在声明中至少找到其中一个值。 此属性在 MicroProfile JSON Web Token 1.2 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.verify.audiences=conferenceService

mp.jwt.verify.issuer

指定要针对 MicroProfile JWT 验证的预期 iss 声明值。 此属性在 MicroProfile JSON Web Token 1.1 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.verify.issuer=https://example.com/issuer

mp.jwt.verify.publickey

以 PKCS8， PEM 或 JWK (S) 格式指定 MicroProfile JWT 签署者的公用密钥的嵌入式密钥材料。 如果找不到任何值，那么将检查 mp.jwt.verify.publickey.location 值。 此属性在 MicroProfile JSON Web Token 1.1 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.verify.publickey=MIIEvgIBADANBgkqh…​

mp.jwt.verify.publickey.algorithm

指定 MicroProfile JWT 端点支持的公用密钥签名算法。 缺省值为 RSA256。 此属性在 MicroProfile JSON Web Token 1.2 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.verify.publickey.algorithm=ES256

mp.jwt.verify.publickey.location

指定公用密钥的相对路径或完整 URL。 将在使用 ClassLoader.getResource 方法的归档中解析所有相对路径。 如果值为 URL ，那么将使用新的 URL(“…”).openStream() 方法对其进行解析。 此属性在 MicroProfile JSON Web Token 1.1 和更高版本中可用。

对受保护资源的每个请求。

mp.jwt.verify.publickey.location=publicKey.pem

MicroProfile REST 客户机

在启用 MicroProfile Rest Client 时，可以指定以下 MicroProfile 配置属性。 有关更多信息，请参阅 MicroProfile Rest Client 规范。

MicroProfile Rest Client 的 MicroProfile 配置属性
名称 描述 当运行时读取属性时 示例

client interface name/mp-rest/connectTimeout

指定等待连接到远程端点的超时值 (以毫秒计)。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/connectTimeout=25

client interface name/mp-rest/hostnameVerifier

指定定制 HostnameVerifier 实例以确定是否允许在 URL 主机名和服务器标识主机名不匹配上失败的 SSL 连接。 该类必须具有公用无参数构造函数。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/hostnameVerifier=myVerifierClass

client interface name/mp-rest/followRedirects

指定一个布尔值，用于确定客户机是否遵循 HTTP 重定向响应。 除 true 以外的任何值都将解释为 false。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/followRedirects=true

client interface name/mp-rest/keyStore

指定密钥库位置，可以是类路径或文件。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/keyStore=file:/home/user/client-keystore.p12

client interface name/mp-rest/keyStorePassword

指定密钥库的密码。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/keyStorePassword=myPassword

client interface name/mp-rest/keyStoreType

指定密钥库类型。 缺省值为 JKS。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/keyStoreType=pcks12

client interface name/mp-rest/providers

指定要包含在客户机中的标准提供程序类名 (相当于 register 方法或 @RegisterProvider 注释) 的逗号分隔列表。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/providers=myClass1,myClass2

client interface name/mprest/providers/interface name/priority

覆盖指定接口的提供程序的优先级。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/providers/com.mycompany.MyProvider/priority=3

`client interface name/mp-rest/proxyAddress

以 proxy : port 格式指定一个字符串值，用于定义 HTTP 代理服务器主机名或 IP 地址以及请求此客户机使用的端口。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/proxyAddress=http://exampleproxy:9041

client interface name/mp-rest/queryParamStyle

指定枚举类型 23 字符串值，用于确定使用同一查询参数的多个值的格式。 可能的值为 MULTI_PAIRS COMMA_SEPARATED或 ARRAY_PAIRS。 缺省值为 MULTI_PAIRS。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/queryParamStyle=ARRAY_PAIRS

client interface name/mp-rest/readTimeout

指定等待来自远程端点的响应的超时值 (以毫秒计)。

在创建 RestClient Bean 时处理方法调用时。

myClient/mp-rest/readTimeout=25

client interface name/mp-rest/scope

指定要用于注入的 CDI 作用域的标准类名。 缺省值为 jakarta.enterprise.context.Dependent。

myClient/mp-rest/scope=jakarta.inject.Singleton

myClient/mp-rest/trustStore=file:/home/user/client-truststore.p12

myClient/mp-rest/trustStorePassword=myPassword

指定要用于此服务的基本 URL。 此属性或 */mp-rest/uri 被视为必需。 但是，某些实现可能具有其他方法来定义这些 URLs/URI。
