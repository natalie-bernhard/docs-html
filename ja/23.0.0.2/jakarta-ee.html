Jakarta EE の概要

Jakarta EE (旧称 Java EE) は、エンタープライズ・アプリケーションを構築するためのオープン・ソースの Java プラットフォームです。 Open Liberty は、トランザクション管理、セキュリティー、データベース・アクセスなどの共通サービスの仕様を提供する Jakarta EE を実装します。

Jakarta EE は、セキュアでスケーラブルなアプリケーションを開発および実行するためのアプリケーション・プログラミング・インターフェース (API) およびランタイム環境を指定します。 API は、アプリケーションがオペレーティング・システムまたは別のアプリケーションまたはサービスの特定のデータまたは機能を使用できるようにするインターフェースです。

Jakarta EE は、アプリケーション・コンポーネントを管理するためのコンテナー・モデルも定義します。 Open Libertyなどの Jakarta EE アプリケーション・サーバーは、これらのコンテナーをホストし、Jakarta EE 仕様用に開発されたすべてのアプリケーションが使用できる標準化されたシステム・サービスを提供します。 これらのオープン・ソース仕様は、さまざまな製品や環境に適用できる構成標準を提供することで、アプリケーションの移植性と回復力を向上させます。 これらの標準仕様は、開発者が低レベルのインフラストラクチャーの詳細を管理するのではなく、アプリケーションの作成に集中できるように支援します。 Jakarta EE は、銀行、e-コマース、旅行などのエンタープライズ・アプリケーションに広く使用されています。

Java SE および Jakarta EE

Jakarta EE は、以前は Oracle Corporation によって所有されており、Java、 Enterprise Edition (Java EE) と呼ばれていました。 Oracle は、 Java EE を 2017 年にオープン・ソースの Eclipse ファウンデーションに移動しました。この時点で、その名前は Jakarta EE に変更されています。 Jakarta EE は、コミュニティー主導のオープン・ソース・モデルを提供し、最新アプリケーションのニーズに素早く対処するために Java EE よりも頻繁にリリースされます。

Jakarta EE は、中核となる Java 機能を提供する Java SE を拡張します。 Java SE は、Java アプリケーションの実行に使用される Java Runtime Environment (JRE) と、Java アプリケーションを開発するためのツールの集合である Java Software Development Kit (SDK) を提供します。 JRE は、主に Java 仮想マシン (JVM) と Java クラス・ライブラリー (JCL) で構成されています。

JVM は、Java アプリケーション・コードを解釈するプログラムです。 Jakarta EE と同様に、JVM は仕様です。 ジャカルタ EE 仕様のさまざまな実装 ( Open Libertyなど) が使用可能であるのと同様に、JVM 仕様のさまざまな実装 ( OpenJ9 や HotSpotなど) が存在します。 これらの各実装は同じ JVM 仕様から派生していますが、それらの間でパフォーマンスの違いが生じる可能性があります。 例えば、 OpenJ9 では、 HotSpotよりも Open Liberty の方がメモリー・フットプリントが良く、起動時間が短縮されます。

JCL は、JVM が実行時に呼び出すことができる、動的にロード可能なライブラリーのセットです。 これらのライブラリーは、最新のオペレーティング・システムで提供される多くの共通関数を提供することにより、Java アプリケーションをオペレーティング・システムから独立させることができます。

以下の図は、Jakarta EE プラットフォームと Java SE プラットフォームの間の関係を示しています。 Java SE プラットフォームは、Java SDK、JRE、JVM、および JCL の仕様を提供します。 Jakarta EE は、エンタープライズ・アプリケーション開発用に設計された API のコレクションを指定することで、Java SE を拡張します。 Open Liberty サーバーは、これらのプラットフォームが指定する機能を実装することによって、クラウド・ネイティブ・アプリケーションをサポートします。

Jakarta EE Platform

Platform のバージョン

Java SE は、より多くのクラスが JCL に追加され、新しい API が導入され、いくつかの古い機能が非推奨になり、置き換えられたため、一連のバージョンを通して進化しました。 現在、長期サポート (LTS) で維持されている Java SE のバージョンは、Java SE 8、Java SE 11、および Java SE 17 のみです。 LTS リリース間で、 Oracle は Java 16 などの短期サポート・リリースを導入しています。 これらの短期リリースは、6 カ月間のみサポートされます。 どのバージョンの Java SE Open Liberty を実行するかについて詳しくは、 Java SE サポートを参照してください。

Java EE 8 は、最初にプラットフォームを Jakarta EE 8 としてリリースした Eclipse ファウンデーションに移行する前の、 Oracle Corporation によるプラットフォームの最後のリリースでした。 ジャカルタ EE 9 は 2020 年に到着した。 このリリースでの主な変更点は、Jakarta パッケージ名の javax を jakarta に置き換えることでした。 2021 年 2 月、Jakarta EE 9.1 がリリースされました。 このリリースには、バージョン 9.0からの API の変更は含まれていませんでしたが、Java SE 11 のサポートが追加されました。

Jakarta EE 仕様

Jakarta EE プラットフォームの継続的な開発は、 Jakarta EE Specification Process (JESP) を通じて、Jakarta 開発者コミュニティーによって進められます。これにより、プラットフォームの新しい仕様プロジェクトを導入するための青写真が提供されます。 Jakarta EE と互換性のあるランタイム ( Open Libertyなど) は、仕様プロジェクトが記述する機能を実装します。 各実装は、 Technology Compatibility Kit (TCK) を使用してコミュニティーによって検証されます。 TCK は、参照実装の準拠性を検査するテストのバッテリーです。 仕様プロジェクトが公式仕様となるためには、TCK テストを満たす少なくとも 1 つのオープン・ソース参照実装を開発するプロセスを通じて承認される必要があります。

Open Liberty は Jakarta EE 9.1の参照実装でした。これは、Jakarta EE Platform Specification で定義されている、必要なすべての Jakarta EE 仕様を実装することを意味します。 Open Liberty で実行されるアプリケーションは、サーバー構成で対応する Open Liberty フィーチャー を使用可能にすることで、Jakarta EE API を利用できます。 以下の表では、最も一般的に使用される Jakarta EE 仕様のいくつかについて説明し、対応する Open Liberty フィーチャーをリストします。 「Try it out」 列には、 Open Liberty が仕様を実装する方法を示す Open Liberty ガイドへのリンクが用意されています。

Jakarta EE および Open Liberty

Open Liberty チームの Jakarta EE への関与は、プロジェクトの先頭に戻ります。 このチームは、 Jakarta Working Group全体にわたって活動しており、Specification Committee や Steering Committee などの委員会に属しています。 開発者は、 Jakarta Batch、 Jakarta Connectors、 Jakarta Concurrency、 Jakarta XML Web Servicesなど、多くの Jakarta EE 仕様のコミッターです。 また、 Jakarta EE Platform のアンブレラ仕様でも主導的な役割を果たしています。

新しい仕様バージョンが導入されると、 Open Liberty はそれらをその モジュラー・フィーチャーの新しいバージョンで実装します。 Open Liberty ゼロ・マイグレーション・アーキテクチャー を使用すると、アプリケーションのニーズに応じて、同じバージョンの API を引き続き使用したり、新しいバージョンにアップグレードしたりする柔軟性が得られます。 Jakarta EE 実装フィーチャーの新規バージョンは、完全な一般出荷版 (GA) リリースに組み込まれる前に、 Open Liberty ベータ・リリース で定期的に導入されます。

Jakarta EE Full Platform および Web Profile

Jakarta EE 仕様は、フル・プラットフォームの Enterprise Editionを定義します。 Web Profile は、フル・プラットフォームの適切に定義されたサブセットです。

Web Profile

Web プロファイルは、最新の Web アプリケーションをターゲットとする、適度に完全なスタックを定義します。 このスタックは、ほとんどの Web アプリケーションのニーズに対応できるフル・プラットフォーム標準 API のサブセットです。

フル・プラットフォーム

フル・プラットフォームは、Jakarta EE プログラミング・モデルの完全な補数を定義します。 Web プロファイル機能に加えて、フル・プラットフォームには、接続、エンタープライズ Bean、メッセージング、およびアプリケーション・クライアントなど、企業が必要とする高度なビジネス機能の仕様があります。

Open Liberty とその使用可能なフィーチャーをインストールする際には、以下の定義を考慮してください。 各インストール・オプションは、 Open Liberty カーネルと、フル・プラットフォームまたは Web Profile をサポートするフィーチャーを提供します。 あるいは、既存の Open Liberty カーネル・インストール済み環境に個別の Jakarta EE フィーチャーをインストールすることもできます。

また、アプリケーションを Open Liberty サーバーにデプロイする際にも、これらの定義を考慮してください。 アプリケーションでエンタープライズ接続 API のサポートが必要な場合は、フル・プラットフォーム機能をインストール済み環境に追加するか、フル・プラットフォーム機能を備えた Open Liberty インストール済み環境にアプリケーションをデプロイする必要があります。

Jakarta EE のサポートを迅速に追加するには、 server.xml ファイルで Web Profile または Jakarta EE Platform コンビニエンス・フィーチャーを指定します。 アプリケーション・クライアント・コンポーネントを素早く構成するために、3 番目の便利な機能である Jakarta EE Application Clientも使用可能です。 コンビニエンス・フィーチャーについて詳しくは、 コンビニエンス・フィーチャーを参照してください。
