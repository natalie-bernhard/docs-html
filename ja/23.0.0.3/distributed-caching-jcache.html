JCache を使用した分散キャッシング

JCache は、データをキャッシュするための標準 Java API です。 Open Liberty は、さまざまなアプリケーション機能 (HTTP セッション・キャッシング、分散認証キャッシング、分散ログアウト Cookie キャッシングなど) での JCache の使用をサポートします。

Open Libertyでは、いくつかのフィーチャーに対して、ローカルのメモリー内キャッシュの代わりに JCache を使用できます。 多数の商用およびオープン・ソースの JCache 実装が使用可能です。 キャッシュを複数の Java ランタイム環境 (JRE) で分散および共有できるようにする実装により、共通データを共有するサーバーのグループ間でパフォーマンスを向上させることができます。

Open Liberty

Open Liberty JCache 構成は、3 つの構成エレメントとその関連属性で構成されます。

cachingProvider

cachingProvider エレメントは、単一の javax.cache.spi.CachingProvider インスタンスを構成します。このインスタンスは、構成された CacheManager オブジェクトのライフサイクルを作成、要求、および後で管理するためのメカニズムを提供します。 cachingProvider エレメントの属性は、 javax.cache.Caching.getCachingProvider メソッドのパラメーターに対応します。 cachingProvider エレメント内のすべての構成済みライブラリーから構成される統合された ClassLoader インスタンスを使用して、 CachingProvider インスタンスがロードされます。 詳しくは、 cachingProviderを参照してください。

cacheManager

cacheManager エレメントは、単一の javax.cache.CacheManager インスタンスを構成します。このインスタンスは、固有の名前を持つキャッシュを確立、構成、獲得、クローズ、および破棄する手段を提供します。 cacheManager エレメントの属性と子エレメントは、 java.util.Properties-javax.cache.spi.CachingProvider#getCacheManager メソッドのパラメーターに対応しています。 詳しくは、 cacheManagerを参照してください。

キャッシュ

cache エレメントは、単一の javax.cache.Cache インスタンスを構成します。これは、アプリケーション・データの一時ストレージを提供するマップのようなデータ構造です。 指定された javax.cache.CacheManager インスタンスに name 属性の値を持つキャッシュが存在しない場合、 Open Liberty はキャッシュを作成しようとします。 詳しくは、 キャッシュを参照してください。

Open Liberty

以下の例は、 Open Libertyでの JCache キャッシュの最小構成を示しています。これには、 cachingProvider、 cacheManager、および cache エレメントのサンプル値が含まれます。 Open Liberty の最小構成は、すべての JCache プロバイダーに対して十分ではない可能性があります。

<library id="JCacheLib">
    <file name="${shared.resource.dir}/jcacheprovider.jar"/>
</library>

<cache id="SampleCache" name="SampleCache">
    <cacheManager>
        <cachingProvider jCacheLibraryRef="JCacheLib" />
    </cacheManager>
</cache>

この例では、 library エレメントは JCache プロバイダー・ファイルの場所を指定します。 cacheManager エレメントは、構成されたライブラリーを参照する cachingProvider エレメントを指定します。 cache エレメントは、 cacheManager エレメントを囲み、JCache キャッシュを使用するアプリケーション・コンポーネントから構成を参照できる名前と ID の値を提供します。

JCache プロバイダー実装および詳細の構成

一部のプロバイダーおよびキャッシング・トポロジーでは、より複雑な構成が必要です。 以下の例では、どの javax.cache.spi.CachingProvider 実装を使用するかを明示的に選択するために、 cachingProvider エレメントで providerClass 属性が定義されています。 プロバイダー固有の構成をプロバイダー定義の URI の形式で渡すために、 cacheManager エレメントで uri 属性も定義されます。 さらに、プロバイダー固有のプロパティーが cacheManager エレメントで定義されます。

<library id="JCacheLib">
    <file name="${shared.resource.dir}/jcacheprovider.jar"/>
</library>

<cache id="SampleCache" name="SampleCache">
    <cacheManager
        uri="file:///${shared.resource.dir}/jcacheconfig.xml" >

        <properties
            org.acme.jcache.prop1="value1"
            org.acme.jcache.prop2="value2" />

        <cachingProvider
            providerClass="org.acme.CachingProvider"
            jCacheLibrarybraryRef="JCacheLib" />
    </cacheManager>
</cache>

providerClass 属性が指定されていない場合、デフォルトの javax.cache.spi.CachingProvider インスタンスが使用されます。 デフォルトの javax.cache.spi.CachingProvider インスタンスを判別するには、JCache プロバイダー JAR ファイル内の META-INF/services/javax.cache.spi.CachingProvider ファイルを確認します。

JCache を使用する複数のコンポーネントの構成

複数のコンポーネントで JCache を使用する場合は、構成に複数の JCache キャッシュ・インスタンスを定義する必要が生じることがあります。 この場合、 cache エレメント内に cacheManager 構成をネストしないでください。 代わりに、 cacheManager 構成で id 属性を定義し、それを cache エレメントの cacheManagerRef 属性から参照してください。 以下の例は、同じ CacheManager インスタンスを参照する 2 つのキャッシュを持つ構成を示しています。

<library id="JCacheLib">
    <file name="${shared.resource.dir}/jcacheprovider.jar"/>
</library>

<cache id="io.openliberty.cache.authentication" name="io.openliberty.cache.authentication"
    cacheManagerRef="CacheManager" />
<cache id="io.openliberty.cache.loggedoutcookie" name="io.openliberty.cache.loggedoutcookie"
    cacheManagerRef="CacheManager" />

<cacheManager id="CacheManager">
    <cachingProvider jCacheLibraryRef="JCacheLib" />
</cacheManager>
