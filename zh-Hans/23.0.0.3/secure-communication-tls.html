使用传输层安全性 (TLS) 进行安全通信

传输层安全性 (TLS) 是一种安全协议，它使用经过验证的证书和加密密钥来保护因特网上的通信。 TLS 是对安全套接字层 (SSL) 协议的更新。

您可以在 Open Liberty 服务器上启用 TLS ，以保护与服务器上的应用程序之间的通信。 TLS 有助于确保敏感数据 (例如用户密码，信用卡号和其他个人信息) 可以在客户机和服务器之间安全地传输。 它也是认证访问 Web 站点的重要协议。 保护认证令牌 (例如，使用 TLS 的 JSON Web 令牌) 的传输，有助于确保用户凭证在不同应用程序和服务之间传递时不会公开。

TLS 协议

TLS 为因特网通信提供端到端认证，机密性和数据完整性。 在客户机与服务器之间发送任何数据之前， TLS 通过交换非对称加密密钥和数字签名证书来确认各方都是他们声称的对象。 在 TLS 握手期间，客户机和服务器商定仅用于该会话的加密算法和共享密钥。 传输的任何数据都将使用该算法和密钥进行加密，只有通信方才能访问该算法和密钥。 此外， TLS 保护的消息通过创建数字签名来建立数据完整性，接收方使用该数字签名来验证消息在传输期间未被篡改。

TLS 握手

TLS 握手是一个过程，客户机和服务器通过该过程在连接期间设置用于安全通信的术语。 在交换任何数据之前发生握手。 通过此过程，客户机和服务器商定 TLS 协议的版本，选择称为密码套件的密码算法，并通过交换和验证数字证书来相互认证。 这些数字证书是包含密钥和有关客户机或服务器的其他标识信息的数据文件。 TLS 证书由认证中心进行数字签名，该认证中心充当可信的第三方，可验证正在交换信息的各方的身份。

如果一方或双方不识别 TLS 证书，那么握手将失败，并且不会建立安全连接。 握手可能由于不同原因而失败，例如证书已到期，或者客户机与服务器所使用的 TLS 版本不匹配。

TLS 和 Open Liberty

要为 Open Liberty 服务器配置 TLS ，请启用 传输安全性 功能部件。 您可以配置此功能，以指定如何为应用程序实现 TLS 握手的详细信息以及 TLS 协议的其他方面。

您还可以通过运行 securityUtility createSSLCertificate 命令为 Open Liberty 服务器创建和配置 TLS 证书。

使用 OCSP 执行 TLS 证书撤销检查

联机证书状态协议 (OCSP) 是 Web 浏览器用于确定 TLS 证书是有效还是已撤销的因特网协议。 它为认证中心提供了一种标准化方法来共享任何已发放证书的状态，以便可以由 Web 服务器或 Web 浏览器客户机对其进行验证。

Open Liberty 不会检查证书撤销，而是依赖于底层 Java 软件开发包 (SDK) 来处理证书撤销检查。 您可以通过设置 Java 安全属性来配置 SDK 以检查证书撤销。 有关更多信息，请参阅 OCSP 支持 PKI的 Java SE 文档。

TLS 和 SSL

在广泛采用 TLS 协议之前， SSL 是保护 Web 通信的标准协议。 随着时间的推移，针对 SSL 识别了许多安全漏洞，并且该漏洞不再被广泛使用。 但是，由于历史原因，术语 "SSL" 通常用于引用加密网络连接，即使在使用 TLS 时也是如此。 在 Open Liberty中，在功能部件和配置名称中仍使用术语 SSL ，即使 TLS 是底层协议也是如此。 实际上， SSL 现在是 TLS 的同义词。
