MicroProfile Config プロパティー

MicroProfile Config は、コンテキストおよび依存関係の注入 (CDI) を使用して、構成プロパティー値をアプリケーションに直接注入する API です。アプリケーション・コードで取得する必要はありません。 MicroProfile Config プロパティーを使用して、他の MicroProfile API の構成値を指定することができます。

MicroProfile Config について詳しくは、 マイクロサービスの外部構成 および Open Liberty MicroProfile Config のガイドを参照してください。

以下のセクションでは、さまざまな MicroProfile API で使用可能な MicroProfile Config プロパティーをリストして説明している表を示します。

MicroProfile コンテキスト伝搬

以下の MicroProfile 構成プロパティーは、 MicroProfile コンテキスト伝搬が有効な場合に指定できます。 詳しくは、 MicroProfile Context Propagation specificationを参照してください。

MicroProfile MicroProfile コンテキスト伝搬の構成プロパティー
名前 説明 ランタイムがプロパティーを読み取るとき 例

mp.context.ManagedExecutor.cleared

ManagedExecutor クラスの cleared パラメーターのデフォルト値を指定します。

ManagerExecutor.builder() クラスが ManagedExecutor インスタンスの作成に使用され、ビルダーがクリアされたコンテキスト・タイプのセットを指定していない場合。

mp.context.ManagedExecutor.cleared=Remaining

mp.context.ManagedExecutor.maxAsync

ManagedExecutor クラスの maxAsync パラメーターのデフォルト値を指定します。

ManagedExecutor インスタンスの作成に ManagerExecutor.builder() クラスが使用され、ビルダーが maxAsync 属性を指定していない場合。

mp.context.ManagedExecutor.maxAsync=10

mp.context.ManagedExecutor.maxQueued

ManagedExecutor クラスの maxQueued パラメーターのデフォルト値を指定します。

ManagedExecutor インスタンスの作成に ManagerExecutor.builder() クラスが使用され、ビルダーが maxQueued 属性を指定していない場合。

mp.context.ManagedExecutor.maxQueued=1

mp.context.ManagedExecutor.propagated

ManagedExecutor クラスの propagated パラメーターのデフォルト値を指定します。

ManagerExecutor.builder() クラスが ManagedExecutor インスタンスのビルドに使用され、ビルダーが伝搬されたコンテキスト・タイプのセットを指定していない場合。

mp.context.ManagedExecutor.propagated=Security,Application

mp.context.ThreadContext.cleared

ThreadContext クラスの cleared パラメーターのデフォルト値を指定します。

ThreadContext.builder() クラスが ThreadContext インスタンスの作成に使用され、ビルダーがクリアされたコンテキスト・タイプのセットを指定していない場合。

mp.context.ManagedExecutor.cleared=Security,Transaction

mp.context.ThreadContext.propagated

ThreadContext クラスの propagated パラメーターのデフォルト値を指定します。

ThreadContext.builder() クラスが ThreadContext インスタンスのビルドに使用され、ビルダーが伝搬されたコンテキスト・タイプのセットを指定していない場合。

mp.context.ThreadContext.propagated=None

mp.context.ThreadContext.unchanged

ThreadContext クラスの unchanged パラメーターのデフォルト値を指定します。

ThreadContext.unchanged() クラスが ThreadContext インスタンスのビルドに使用され、ビルダーがクリアされた未変更タイプのセットを指定していない場合。

mp.context.ManagedExecutor.unchanged=Remaining

MicroProfile フォールト・トレランス

以下の MicroProfile Config プロパティーは、 MicroProfile フォールト・トレランスが有効な場合に指定できます。 詳しくは、 MicroProfile Fault Tolerance specificationを参照してください。

MicroProfile MicroProfile フォールト・トレランスの構成プロパティー
名前 説明 ランタイムがプロパティーを読み取るとき 例

classname/methodname/annotationname/parametername=newvalue

フォールト・トレランス・アノテーションに設定されている値をメソッド・レベルでオーバーライドします。

MicroProfile Fault Tolerance バージョン 3.0 以降では、このプロパティーは、メソッドの初回実行時に、Bean クラスごとにメソッドごとに 1 回読み取られます。 MicroProfile Fault Tolerance バージョン 2.1 以前では、メソッドの初回実行時に、Bean インスタンスごとにメソッドごとに 1 回読み取られます。

com.acme.ClassA/methodB/Retry/maxRetries=5

classname/annotationname/parametername=newvalue

フォールト・トレランス・アノテーションに設定されている値をアノテーション・レベルでオーバーライドします。

MicroProfile Fault Tolerance バージョン 3.0 以降では、このプロパティーは、メソッドの初回実行時に、Bean クラスごとにメソッドごとに 1 回読み取られます。 MicroProfile Fault Tolerance バージョン 2.1 以前では、メソッドの初回実行時に、Bean インスタンスごとにメソッドごとに 1 回読み取られます。

com.acme.ClassA/Retry/maxRetries=5

mp.fault.tolerance.interceptor.priority

Fault Tolerance インターセプターの基本優先度を構成します。 このプロパティーは、 MicroProfile Fault Tolerance バージョン 2.0 以降で使用可能です。

アプリケーションの始動

mp.fault.tolerance.interceptor.priority=1000

MP_Fault_Tolerance_Metrics_Enabled

フォールト・トレランスによって生成されるすべてのメトリックを無効または有効にします。 デフォルトは true です。 このプロパティーは、 MicroProfile Fault Tolerance バージョン 1.1 以降で使用可能です。

アプリケーションの始動
