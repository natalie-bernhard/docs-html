Open Liberty 的 JDBC 驱动程序跟踪

您可以在 Open Liberty 中对第三方 JDBC 驱动程序启用跟踪，以用于调试和支持目的。 要收集 JDBC 驱动程序跟踪，必须指定 JDBC 驱动程序，设置日志记录级别，并提供任何特定于驱动程序的必需配置。

了解 Open Liberty 跟踪

如何对 JDBC 驱动程序启用跟踪取决于该驱动程序是否支持 java.util.logging Java 日志记录库或其他一些日志记录库，例如 SLF4J 或 Log4J。 在任一情况下，都必须在跟踪规范中标识您的驱动程序。 支持 java.util.logging 库的驱动程序由驱动程序使用的软件包名称指定。

不支持 java.util.logging 库的驱动程序由 JDBC 驱动程序的日志写程序名称指定。 Open Liberty 为基于 JDBC 规范方法 (例如 getConnection 和 executeQuery) 生成日志的公共数据库供应商提供日志写程序。 Open Liberty 还为不支持 java.util.logging 库且不存在特定于驱动程序的日志写程序的驱动程序提供通用日志写程序。

为驱动程序启用跟踪时，可以 将定制跟踪设置配置为供应商属性。

有关 Open Liberty 跟踪的更多信息，请参阅 日志和跟踪配置。

对 JDBC 驱动程序启用跟踪

以下示例演示如何对 JDBC 驱动程序启用跟踪。 有关包含公共 JDBC 驱动程序的必需包或日志写程序名称的特定于驱动程序的示例，请参阅 公共 JDBC 驱动程序的跟踪配置示例。

对于支持 java.util.logging的 JDBC 驱动程序，请指定该驱动程序使用的程序包名，如 Microsoft SQL JDBC 驱动程序的以下 server.xml 文件示例中所示。

<logging traceSpecification="*=info:RRA=all:com.microsoft.sqlserver.jdbc=all" maxFiles=10 maxFileSize=100 />

此示例还指定可选的 maxFiles 和 maxFileSize 属性，这将配置日志中文件大小和数量的限制。 有关这些属性和其他 logging 元素属性的更多信息，请参阅 日志记录。

对于不支持 java.util.logging 库的 JDBC 驱动程序，请指定特定于驱动程序的日志写程序的名称或通用 Open Liberty 日志写程序。 以下 server.xml 文件示例显示了 Derby JDBC 驱动程序的跟踪规范，该驱动程序不支持 java.util.logging 库。

<logging traceSpecification="*=info:RRA=all:com.ibm.ws.derby.logwriter=all" />

在处理 server.xml 文件之前初始化日志记录。 因此，通过 server.xml 文件配置日志记录可能会导致早期日志条目使用与后来的日志条目不同的日志配置。 要避免此问题，可以改为在 bootstrap.properties 文件中配置跟踪规范。 以下 bootstrap.properties 文件示例显示了 Microsoft SQL Server DataDirect 驱动程序的跟踪规范，该驱动程序不支持 java.util.logging 库。

com.ibm.ws.logging.trace.specification=*=info:RRA=all:com.ibm.ws.sqlserver.logwriter=all
com.ibm.ws.logging.max.files=10
com.ibm.ws.logging.max.file.size=100

还可以在 bootstrap.properties 文件中通过指定驱动程序的包名 (而不是日志写程序名) 来配置支持 java.util.logging 库的驱动程序。

配置定制设置

如果 JDBC 驱动程序具有定制跟踪设置，那么可以将这些定制跟踪设置设置为 server.xml 文件中的 JDBC 驱动程序供应商属性。

<logging traceSpecification="*=info:RRA=all:com.ibm.db2=all" ... />
<datasource ...>
  <!-- traceLevel -1 is equivalent to ALL -->
  <properties.db2.jcc traceLevel="-1"/>
</datasource>

在此示例中， DB2 JCC 驱动程序的定制 traceLevel 属性是使用特定于驱动程序的 properties.db2.jcc 子元素设置的。 有关特定于驱动程序的可用属性的更多信息，请参阅 数据源。

Oracle

Oracle 提供了两个不同的驱动程序，一个用于生产，另一个用于调试目的。 生产驱动程序不会产生任何跟踪，因此您需要下载生产驱动程序并将其替换为调试驱动程序。 调试驱动程序在驱动程序名称中包含 _g 。 例如， ojdbc8.jar 是 ojdbc8_g.jar。 通过在 jdbcDriver 元素中指定 library 子元素来配置调试驱动程序。 library 子元素定义调试驱动程序 JAR 文件的路径。

Oracle 驱动程序支持 java.util.logging 库。 驱动程序使用跟踪规范中的 oracle 包名。 以下 server.xml 文件示例显示如何配置 Oracle 调试驱动程序并启用跟踪。

<logging traceSpecification="*=info:RRA=all:oracle=all" />
<jdbcDriver id="oracleDriver">
  <library id="oracleDebug">
      <file name="path_to_oracle_driver/ojdbcX_g.jar"/>
  </library>
</jdbcDriver>

将以下选项添加到 jvm.options 文件以确保生成所有跟踪。

-Doracle.jdbc.Trace=true

Oracle 定制跟踪

缺省情况下， Oracle 驱动程序跟踪与 Liberty 跟踪结合使用，因为这两个跟踪都使用 java.util.logging 库。 但是， Liberty 还支持将 Oracle 跟踪与 Liberty 跟踪分离。 如果需要，您可以使用此函数向 Oracle 支持人员提供独立 Oracle JDBC 跟踪。

以下 server.xml 文件示例显示如何配置 Oracle 调试驱动程序以生成单独的跟踪。 不需要跟踪规范。

<jdbcDriver id="oracleDriver">
  <library id="oracleDebug">
      <file name="path_to_oracle_driver/ojdbcX_g.jar"/>
  </library>
</jdbcDriver>

其余配置由系统属性提供。 将以下必需属性添加到 jvm.options 文件以启用新的基于文件的记录器。

-Doracle.jdbc.Trace=true
-DoracleLogFileName=<your-log-name>.log
-DoracleLogPackageName=<package-to-trace>

您还可以使用可选属性在 jvm.options 文件中定制已配置的跟踪。

-DoracleLogFileSizeLimit=<size-in-bytes>
-DoracleLogFileCount=<number-of-files>
-DoracleLogFormat=<SimpleFormatter|XMLFormatter|your-fully-qualified-class>
-DoracleLogTraceLevel=<OFF|SEVERE|WARNING|INFO|CONFIG|FINE|FINER|FINEST|ALL>

先前的属性是可选的。 如果未配置任何值，那么 Liberty 将使用以下缺省值:

  • oracleLogFileSizeLimit: 0 (除去大小限制，文件可增长到任何大小)

  • oracleLogFileCount: 1 (所有跟踪都将放入单个日志文件中)

  • oracleLogFormat: SimpleFormatter (来自 java.util.logging的简单格式化程序)

  • oracleLogTraceLevel:INFO

有关 Oracle 建议的设置，请参阅 Oracle 文档。

<logging traceSpecification="*=info:RRA=all:org.postgresql=all" />

在 V 42.0.0之前， PostgreSQL 驱动程序不支持 java.util.logging 库。 Open Liberty 在跟踪规范中提供 com.ibm.ws.postgresql.logwriter 日志写程序名称。 以下 server.xml 文件示例显示了用于对低于 42.0.0的版本中的 PostgreSQL 驱动程序启用跟踪的配置。

<logging traceSpecification="*=info:RRA=all:com.ibm.ws.postgresql.logwriter=all" />

其他数据库

对于不支持 java.util.logging 库且 Open Liberty 未提供特定于驱动程序的日志写程序的驱动程序，将在跟踪规范中使用通用 com.ibm.ws.database.logwriter 日志写程序名称。 以下 server.xml 文件示例显示如何使用通用日志写程序启用跟踪。
