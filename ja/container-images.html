Open Liberty コンテナー・イメージ

コンテナー・イメージは、コンテナー化されたアプリケーションをビルドして実行するために必要なすべてのものを含む、階層化された実行可能ソフトウェア・パッケージです。 Open Liberty コンテナー・イメージを使用して、開発ニーズを満たすアプリケーション・イメージをビルドできます。

Open Liberty コンテナー・イメージは、 IBM Container Registry (ICR) から入手できます。 認証なしで ICR からイメージをプルできます。

IBM Container Registry

Open Liberty コンテナー・イメージ・タグを icr.io/appcafe/open-liberty: ソースに追加して、ICR からプルするコンテナー・イメージの詳細を指定できます。 詳しくは、「 アプリケーション・イメージのビルド 」セクションを参照してください。

ICR の最新の Open Liberty リリースのコンテナー・イメージ・タグを以下から選択します。

  • kernel-slim-java8-openj9-ubi

  • kernel-slim-java11-openj9-ubi

  • kernel-slim-java17-openj9-ubi

  • full-java8-openj9-ubi

  • full-java8-ibmjava-ubi

  • full-java11-openj9-ubi

  • full-java17-openj9-ubi

  • beta

  • latest

これらのタグは、バージョン番号を指定していないため、最新バージョンをプルします。 イメージ・タグには、最新バージョン以外のバージョンを指定できます。 タグ値を指定しない場合、デフォルトで latest が使用されます。 これらのタグ値について詳しくは、「 コンテナー・イメージ・タグの命名規則 」セクションを参照してください。

IBM Cloud ID がある場合は、 IBM Cloud Container Registry CLI プラグイン がインストールされた IBM Cloud CLI を使用して、ICR で現在使用可能なイメージのリストを生成できます。 認証が必要なのは、イメージをリストする場合のみです。 認証なしで ICR からイメージをプルできます。 イメージをリストするには、以下のコマンドを実行します。

ibmcloud login
ibmcloud cr region-set global
ibmcloud cr images -- appcafe/open-liberty を制限します

アプリケーション・イメージのビルド

Dockerfile ファイルまたは Containerfile ファイルで Open Liberty コンテナー・イメージ・タグを使用して、単一のアプリケーションとそれに対応する構成を追加するアプリケーション・イメージをビルドできます。 デバッグ目的以外では、起動後にコンテナを手動で設定しないでください。これらの変更は、イメージから作成される新しいコンテナには反映されないためです。

以下の例のようにイメージ・テンプレートを設定します。この例では、 kernel-slim-java17-openj9-ubi タグを使用して、Java 17 の OpenJ9 ディストリビューションで最新の Open Liberty リリースを使用するイメージをビルドします。

FROM icr.io/appcafe/open-liberty:kernel-slim-java17-openj9-ubi

# Add a Liberty server configuration including all necessary features
COPY --chown=1001:0  server.xml /config/

# This script adds the requested XML snippets to enable Liberty features and grow the image to be fit-for-purpose.
# This option is available only in the 'kernel-slim' image type. The 'full' and 'beta' tags already include all features.
RUN features.sh

# Add interim fixes (optional)
COPY --chown=1001:0  interim-fixes /opt/ol/fixes/

# Add an application
COPY --chown=1001:0  Sample1.war /config/dropins/

# This script adds the requested server configuration, applies any interim fixes, and populates caches to optimize the runtime.
RUN configure.sh

この例のようなテンプレートを使用すると、アプリケーションと構成がプリインストールされたコンテナー・イメージが作成されます。これは、いつでも、完全に構成された新しいコンテナーを作成できることを意味します。 アプリケーション・イメージをカスタマイズするために指定できるさまざまな引数および環境変数について詳しくは、 Open Liberty コンテナー・イメージの GitHub 資料を参照してください。

Open Liberty InstantOn を使用して、アプリケーション・コンテナー・イメージの起動時間を改善することもできます。

コンテナー・イメージ・タグの命名規則

Open Liberty コンテナー・イメージ・タグは、以下の命名規則を使用します。

< オプションのフィックスパック・バージョン-> < liberty イメージ・タイプ>-< java バージョン>-< java タイプ>-ubi

以下のタグ値は、コンテナー・イメージに含まれるリソースを決定します。

フィックスパック・バージョン

この値はオプションです。 フィックスパック・バージョンを指定しない場合、コンテナー・イメージには Open Libertyの最新リリースが含まれます。 最新リリースに加えて、 kernel-slim イメージ・タイプと full イメージ・タイプについては、最新の 2 つの四半期ごとのフィックスパック・リリースが常に維持されます。

Open Liberty の四半期ごとのリリース・バージョンは、 .3、 .6、 .9、または .12で終わります。 例えば、最新の Open Liberty リリースが 22.0.0.10の場合、 22.0.0.9 および 22.0.0.6 リリースも ICR で維持されます。 最新のリリース・バージョンではなく、これらのリリース・バージョンの 1 つをプルするには、選択したリリース・バージョンを前のリストの kernel-slim または full イメージ・タグの先頭に追加します。 例えば、 22.0.0.9 が最新の四半期リリースである場合、 22.0.0.9-full-java8-openj9-ubi タグを指定することにより、Java 8 の OpenJ9 ディストリビューションを使用する 22.0.0.9 リリースを含むイメージをプルできます。 beta イメージは、最新のベータ版でのみ使用可能です。

Liberty イメージ・タイプ

2 つの異なるイメージ・タイプ ( kernel-slim または full) を使用できます。 full タイプは、すべての Open Liberty フィーチャーおよびパッケージがインストールされたサーバーを提供します。

kernel-slim タイプは、最小限のベア・サーバーのみを提供します。これは、アプリケーションが必要とするフィーチャーのみを追加することでカスタマイズできます。 このイメージ・タイプは、実動イメージのベスト・プラクティスをサポートします。これは、アプリケーションに不可欠なもののみをパッケージする、より軽量のイメージになるためです。 詳しくは、「 アプリケーション・イメージのビルド 」セクションを参照してください。

Java バージョン

この値は、ランタイムが使用する Java SE のバージョンを決定します。 タグは、 Java SE サポート・ページにリストされているすべての Java 長期サポート (LTS) リリースで使用できます。

Java 型

この値は、ランタイムが使用する Java SE ディストリビューションのタイプを決定します。 現在のオプションは、 IBM Semeru with Eclipse OpenJ9 、および IBM SDK, Java Technology Edition (Java SE 8 のみ) です。

この命名規則に従うタグに加えて、 Open Liberty には、特定のイメージをプルする 3 つの事前フォーマット済みタグが用意されています。

  • latest: このタグは、Java 8 を使用する最新の Open Liberty リリース全体のプルを単純化します。 これは full-java8-openj9-ubi タグの別名です。 タグ値を指定しない場合、デフォルトで latest が使用されます。

  • beta: このタグは、最新のベータ・リリースを提供します。これには、最新リリースのすべての機能に加えて、現在開発中の新機能と更新された機能が含まれます。
