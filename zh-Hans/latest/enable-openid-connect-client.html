为应用程序启用 OpenID Connect 客户机

您可以在 Web 或 RESTful Web Services (JAX-RS) 应用程序中使用 OpenIdAuthenticationMechanismDefinition 注释来为该应用程序启用 OpenID Connect 客户机。 注释可以通过消除对手动协议实现的需求，简化在 Web 或 JAX-RS 应用程序中启用 OpenID Connect 认证的过程。

OpenIdAuthenticationMechanismDefinition 注释是 Jakarta Security 3.0 规范的一部分，在 Open Liberty 中受 Application Security 5.0 功能部件支持。

配置 OpenID Connect 客户机和回调 servlet

要在应用程序中使用 OpenIdAuthenticationMechanismDefinition 注释，必须在 server.xml 文件中启用 appSecurity-5.0 功能部件。 以下示例显示了具有最小 OpenIdAuthenticationMechanismDefinition 注释的 servlet。

@WebServlet("/OidcAnnotatedServlet")
@OpenIdAuthenticationMechanismDefinition(
                    providerURI = "https://localhost:8920/oidc/endpoint/OP",
                    clientId = "clientId",
                    clientSecret = "clientSecret",
                    redirectURI = "https://localhost:8940/MyApp/Callback")
@DeclareRoles("all")
@ServletSecurity(@HttpConstraint(rolesAllowed = "all"))
public class OidcAnnotatedServlet extends HttpServlet {
…
}

此示例演示如何使用 OpenIdAuthenticationMechanismDefinition 注释对名为 MyApp 的应用程序中的受保护资源启用基本 OpenID Connect 客户机。 此客户机在 https://localhost:8920/oidc/endpoint/OP URL 上使用 OpenID Connect 提供者 (OP)。 当您尝试访问 MyApp 应用程序中的受保护资源时，系统会将您定向到 OP 授权端点，该端点是从 OP 发现元数据中获取的。 认证后， OP 会使用授权代码将您重定向到 https://localhost:8940/MyApp/Callback URL。

OpenID Connect 客户机接收认证响应，对其进行验证，并使用授权代码检索 access， identity和 userinfo 令牌。 然后，在将请求分派到回调 servlet 之前，将验证并使用这些令牌来建立已认证的主体集。 服务器指定的 缺省 TLS 配置 由 OpenID Connect 客户机在向 OP token 和 userinfo 端点发送请求时使用。 因此，必须将 OP 的签署者证书导入到为缺省 TLS 配置配置的缺省信任库中。

要使用最小注释，应用程序需要为回调 servlet 提供实现。 此回调 servlet 必须使用 OpenIdContext API 来检索原始请求 URL ，然后重定向到原始资源，如以下示例中所示。

…
import jakarta.security.enterprise.identitystore.openid.OpenIdContext;
…

@WebServlet("/Callback")
public class CallbackServlet extends HttpServlet {

    @Inject
    private OpenIdContext context;

    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        …
        if (context != null) {
            Optional<String> originalRequest = context.getStoredValue(request, response, OpenIdConstant.ORIGINAL_REQUEST);
            String originalRequestString = originalRequest.get();
            response.sendRedirect(originalRequestString);
        }
    }
}

您可以通过在注释中将 redirectToOriginalResource 属性设置为 true 来避免创建回调 servlet ，如以下示例中所示。 OpenID Connect 客户机会自动重定向到原始资源。

@OpenIdAuthenticationMechanismDefinition(
                    redirectToOriginalResource = true,
                    …

配置定制元数据

您还可以通过向注释添加 providerMetadata 属性来覆盖从 OP 发现端点获取的元数据。 以下示例将覆盖授权和令牌端点。

@OpenIdAuthenticationMechanismDefinition(
                    providerMetadata = @OpenIdProviderMetadata(
                                        authorizationEndpoint = "https://localhost:8920/oidc/endpoint/OP/authorize",
                                        tokenEndpoint = "https://localhost:8920/oidc/endpoint/OP/token"),
                    …

缺省情况下， OpenID Connect 客户机使用 OP 发现元数据中的 jwks_uri 值来获取验证令牌签名所需的签名密钥。 但是，您可以使用 providerMetadata 属性的 @OpenIdProviderMetadata 注释中的 jwksURI 属性来指定其他 JWK 端点 URL。

@OpenIdAuthenticationMechanismDefinition(
                    providerMetadata = @OpenIdProviderMetadata(
                                        jwksURI = "https://localhost:8920/oidc/endpoint/OP/jwk",
                    …

指定表达式语言表达式

所有属性都支持 Jakarta Expression Language 5.0 (EL) ，并且可以在 String 属性中直接指定 EL 表达式。 例如，如果您有提供私钥的 Bean ，并且您不希望将其包含在注释中，那么可以使用以下语法。

@OpenIdAuthenticationMechanismDefinition(
                    clientSecret = "${openIdConfig.clientSecret}",
                    …

您可以将为 EL 表达式设计的备用属性用于非字符串属性。 以下示例中 useSessionExpression 属性中 EL 表达式的布尔结果将覆盖 useSession 布尔属性的值。

@OpenIdAuthenticationMechanismDefinition(
                    useSession = true,
                    useSessionExpression = "#{openIdConfig.useSessionExpression}",
                    …

指定缺失声明

如果 OP 生成缺少用户名的 preferred_username 声明或组的 groups 声明的令牌，那么可以使用 claimsDefinition 属性。 Liberty OP 使用 sub 和 groupIds 声明生成身份令牌。 必须使用 claimsDefinition 属性来指定名称和组声明，尤其是当 providerURI 属性指向 Liberty OP 时。

以下示例显示如何为调用者名称指定 sub 声明以及为组指定 groupIds 声明。

@OpenIdAuthenticationMechanismDefinition(
                    providerURI = "https://localhost:8920/oidc/endpoint/LibertyOP",
                    claimsDefinition = @ClaimsDefinition(callerNameClaim = "sub", callerGroupsClaim = "groupIds"),
                    …

如果没有应用程序的任何角色映射，那么会自动将资源的角色保护从 callerGroupsClaim 属性指定的声明映射到关联组。 例如，受 all 角色保护的应用程序将需要 groupIds 声明中的 all 组。

检查令牌到期时间

通过使用 LogoutDefinition 注释和 logout 属性，您可以检查令牌到期及其相应结果。 以下示例显示如何启用对访问权和身份令牌到期的检查。 它还演示了如何在任一令牌到期时使用 OP end_session_endpoint 激活依赖方 (RP) 发起的注销。

@OpenIdAuthenticationMechanismDefinition(
                    logout = @LogoutDefinition(
                                        notifyProvider = true,
                                        accessTokenExpiry = true,
                                        identityTokenExpiry = true),
                    …

有关 OpenIdAuthenticationMechanismDefinition 注释及其属性的更多信息，请参阅 部分 2.4.4。 OpenID Connect 注释。
