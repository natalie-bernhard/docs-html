Logstash コレクターを使用した Logstash へのログの転送

Open Liberty サーバーからログを収集し、分析およびトラブルシューティングの目的でそれらをダッシュボードに表示することができます。 Logstash Collector フィーチャーは、 Open Liberty からログを収集し、それらをリモート Logstash サーバーに送信します。リモート Logstash サーバーでは、 Elasticsearch や Kibana などの製品によって管理および視覚化することができます。

Logstash は、分析のためにログ・データを準備するオープン・ソースのログ管理ツールです。 Logstash は、データを処理して変換し、 Elasticsearch、 Kafka、 PagerDuty、または E メール・サーバーなどのさまざまな出力に送信することができます。

Logstash Collector フィーチャーを使用するタイミング

Open Liberty サーバーが Kubernetes クラスターなどの管理対象環境にデプロイされている場合、ログは通常、その環境内のエージェントによって集約されます。 この場合、Logstash Collector フィーチャーを使用する必要はありません。

サーバー・ログが外部エージェントによって管理されていない場合は、ログを Logstash に転送するように Filebeat を構成できる可能性があります。 Open Liberty サーバーと同じホストに Filebeat をインストールできる場合、JSON 形式でログを書き込むように Open Liberty を構成でき、 Filebeat はログを転送できます。 詳しくは、 Elastic スタックを使用した JSON ログの分析を参照してください。

Open Liberty サーバーと同じホストに Filebeat をインストールできない場合は、 Open Liberty Logstash コレクター ・フィーチャーを使用してログを Logstash に転送できます。 この機能は、実行時にログ・データを収集し、Logstash サーバーに送信します。

この機能は、 Logstash から使用可能な出力プラグインのいずれかによって分析のためにログ・データを送信できますが、多くのユーザーは、 Elasticsearch および Kibana で Logstash を使用することを選択します。 これらの 3 つのツールは、ログ・データを収集、管理、および視覚化するために広く一緒に使用されています。 詳しくは、 Elastic の Web サイトを参照してください。

Logstash Collector フィーチャーは、以下の製品でテスト済みです。

  • Logstash V5.3.x、Elasticsearch V5.3.x、および Kibana V5.3.x

  • Logstash V6.4.x、Elasticsearch V6.4.x、および Kibana V6.4.x

  • Logstash V7.x、Elasticsearch V7.x、および Kibana V7.x

始める前に

Elastic からの指示に従って、 Logstash、 Elasticsearch、および Kibana をセットアップします。

Logstash Collector フィーチャーを使用してログを転送するためのサーバーの構成

以下の手順は、Logstash Collector フィーチャーを使用して Open Liberty ログ・データを収集する方法を示しています。 その後、それを Logstash サーバーに送信し、 Elasticsearchに保管し、Kibana ダッシュボードで表示することができます。

  1. 証明書および鍵ペア・ファイルを作成または取得して、Logstash の SSL 接続または TLS 接続を有効にします。
    以下の OpenSSL のコマンド例は、証明書と鍵ペアを生成します。 必要に応じて、鍵が有効である日数をカスタマイズします。

    openssl req -x509 -newkey rsa:2048 -keyout logstash.key -out logstash.crt -days 365 -nodes

  2. Sample Logstash Collector dashboards for Liberty repositoryから、サンプル Logstash 構成ファイルと索引テンプレート・ファイルをダウンロードします。

    1. liberty_logstash.conf Logstash 構成ファイルおよび liberty_logstash_template_ibm.json 索引テンプレート・ファイルを、ご使用の Elastic バージョンに対応するディレクトリーからダウンロードします。

    2. liberty_logstash.conf ファイルで、以下のステップを実行します。

      • ステップ 1 で生成した証明書と鍵ペアのファイルを使用して、Beats の ssl_certificate パスと ssl_key パスをカスタマイズします。

      • Elasticsearch 構成のホスト名とポート番号を使用して、 Elasticsearch_host_name:port_number Elasticsearch ホスト値をカスタマイズします。

      • liberty_logstash_template_ibm.json 索引テンプレート・ファイルへのパスを使用して、 Elasticsearch テンプレート・パスをカスタマイズします。

  3. イベントを収集する Open Liberty サーバーごとに、以下のステップを実行します。

    1. Open Liberty サーバー用の鍵ストアを取得または作成します。 自己署名証明書を作成するには、以下の例に示すように、 wlp_install_dir/bin フォルダーから securityUtility コマンドを実行します。 必要に応じて、サーバー名、パスワード、およびサブジェクトをカスタマイズします。

      d: ¥ wlp¥ bin¥securityUtility createSSLCertificate --server=server1 -- password="Liberty " --subject=CN=myHostname,OU=server1, O=ibm, C=us

      key.p12 デフォルト鍵ストア・ファイルは、 wlp_install_dir/usr/servers/server1/resources/security ディレクトリーの下に作成されます。 このコマンドからの出力は以下の例のようになります。

      Creating keystore wlp_install_dir/usr/servers/server1/resources/security/key.p12

Created SSL certificate for server server1. The certificate is created with CN=myHostname,OU=server1,O=ibm,C=us as the SubjectDN.

Add the following lines to the server.xml to enable SSL:

    <featureManager>
        <feature>ssl-1.0</feature>
    </featureManager>
    <keyStore id="defaultKeyStore" password="{xor}EzY9Oi0rJg==" />

      コマンド出力からのサーバー構成スニペットを server.xml ファイルに追加します。

    2. Java Developer Kit の keytool コマンドを実行して、ステップ 1 で作成した logstash.crt ファイルをサーバー内の key.p12 ファイルにインポートします。 以下の例では、必要に応じて wlp_install_dir ディレクトリーおよびサーバー名をカスタマイズします。 パスワードを求めるプロンプトが出されたら、ステップ 3a: で指定した証明書パスワードを使用します。

      keytool -importcert -noprompt -alias logstash -file logstash.crt -keystore wlp_install_dir/usr/servers/server1/resources/security/key.p12 -storepass Liberty

    3. Logstash Collector フィーチャーがまだサーバーにインストールされていない場合は、以下のコマンドを使用してインストールします。

      d: ¥ wlp¥ bin¥featureUtility installFeature logstashCollector-1.0

    4. server.xml ファイルで Logstash Collector フィーチャーを構成 します。

    5. 「HTTP アクセス・ロギング」を有効にします。

  4. Elasticsearch、Logstash、および Kibana を開始します。 手順については、 Elastic の Web サイト を参照してください。

  5. Open Liberty サーバーを始動し、いくつかのイベントを生成します。

Logstash Collector 機能を使用してリモート Logstash サーバーにログを送信するようにサーバーを構成しました。 Liberty のサンプル Kibana ダッシュボードを使用して、ログの視覚化を作成できるようになりました。

ログを視覚化するための Kibana ダッシュボードのセットアップ

Logstash Collector 機能を使用してログを収集および転送するようにサーバーを構成した後、以下のステップを実行して、Kibana ダッシュボードでログ・イベントを表示します。

  1. ブラウザーで Kibana を開き、索引を作成します。

    • Kibana 7、6、または 5.6の場合、 「管理」 > 「索引パターン」をクリックします。

      1. logstash-* を「索引パターン (Index Pattern)」 として入力します。

      2. 「詳細オプション」をクリックし、logstash-* を「索引パターン ID (Index Pattern ID)」 として入力します。

      3. 「時間フィルター」 フィールド名として datetime を選択します。 「作成 (Create)」 をクリックします。

    • Kibana 5.0 - 5.5:

      「管理」 > 「索引パターン」をクリックし、 「時間フィルター」 フィールド名として datetime を選択します。 「作成」をクリックします。

  2. 「Liberty 用のサンプル Logstash コレクター・ダッシュボード (Sample Logstash Collector dashboards for Liberty)」 リポジトリーで、ご使用のバージョンの Elastic に対応するディレクトリーからサンプル・ダッシュボードをダウンロードします。

  3. ダッシュボードを Kibana にインポートします。

    「管理」 > 「保存済みオブジェクト」 > 「インポート」をクリックし、ステップ 2 でダウンロードしたダッシュボードを選択します。

  4. ダッシュボードを表示します。

    「ダッシュボード」 > 「開く」をクリックし、ステップ 3 でインポートしたダッシュボードを選択します。

Logstash Collector フィーチャーを使用して Logstash サーバーにイベントを送信するように Open Liberty サーバーを構成しました。 Kibana を使用して Liberty ダッシュボードでログ・イベントの視覚化を表示できるようになりました。
